## Endpoints

{/* List all endpoints */}

### `GET /resource`

{/* Brief description */}

Retrieves a list of [resources].

#### Parameters

<ParamField query="limit" type="integer" default="20">
  Maximum number of results to return (1-100)
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip for pagination
</ParamField>

<ParamField query="filter" type="string" optional>
  Filter results by specific criteria
</ParamField>

#### Request Example

```bash
curl -X GET "https://api.example.com/v1/resource?limit=10" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

#### Response

<ResponseField name="data" type="array" required>
  Array of resource objects
</ResponseField>

<ResponseField name="pagination" type="object" required>
  Pagination metadata
  <Expandable title="Properties">
    <ResponseField name="total" type="integer">
      Total number of resources
    </ResponseField>
    <ResponseField name="limit" type="integer">
      Current limit parameter
    </ResponseField>
    <ResponseField name="offset" type="integer">
      Current offset parameter
    </ResponseField>
  </Expandable>
</ResponseField>

#### Response Example

```json
{
  "data": [
    {
      "id": "resource-123",
      "name": "Example Resource",
      "status": "active",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "pagination": {
    "total": 150,
    "limit": 10,
    "offset": 0
  }
}
```

### `POST /resource`

Creates a new [resource].

#### Request Body

<ParamField body="name" type="string" required>
  Name of the resource (1-100 characters)
</ParamField>

<ParamField body="config" type="object" required>
  Configuration object
  <Expandable title="Properties">
    <ParamField body="config.setting1" type="string" required>
      Description of setting1
    </ParamField>
    <ParamField body="config.setting2" type="boolean" default="false">
      Description of setting2
    </ParamField>
  </Expandable>
</ParamField>

#### Request Example

```bash
curl -X POST "https://api.example.com/v1/resource" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Resource",
    "config": {
      "setting1": "value1",
      "setting2": true
    }
  }'
```

#### Response

```json
{
  "id": "resource-456",
  "name": "My Resource",
  "config": {
    "setting1": "value1",
    "setting2": true
  },
  "status": "creating",
  "created_at": "2024-01-15T11:00:00Z"
}
```

## Error Codes

{/* Comprehensive error documentation */}

| Status | Code | Description |
|--------|------|-------------|
| 400 | `INVALID_REQUEST` | Request body validation failed |
| 401 | `UNAUTHORIZED` | Authentication required or invalid |
| 403 | `FORBIDDEN` | Insufficient permissions |
| 404 | `NOT_FOUND` | Resource does not exist |
| 409 | `CONFLICT` | Resource already exists or state conflict |
| 429 | `RATE_LIMITED` | Too many requests |
| 500 | `INTERNAL_ERROR` | Server error (contact support) |

### Error Response Format

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "Validation failed",
    "details": [
      {
        "field": "name",
        "message": "Name is required"
      }
    ]
  }
}
```
