2. **Pattern 4: Multi-Agent** — H2 section with:
   - **Definition**: Enterprises create value through teams of specialists. The multi-agent pattern mirrors this by connecting networks of specialized agents—each focused on different workflow stages—under an orchestrator. This modular design enables agility, scalability, and easy evolution while keeping responsibilities and governance clear.
   - **Key capabilities** (bullet list):
     - Connect specialized agents across workflow stages
     - Scale from single agent to agent teams
     - Maintain clear responsibility boundaries
     - Enable modular evolution of agent capabilities
   - **Citadel placement**: Layer 2 (AI Control Plane) provides orchestration traces and evaluations. Layer 1 (Governance Hub / APIM) routes inter-agent communication securely. Layer 3 (Agent Identity) ensures each agent has a valid identity.
   - **Azure services**: Azure AI Foundry Agent Service, Azure Service Bus, Azure API Management
   - **Customer example**: JM Family adopted multi-agent approach with BAQA Genie, deploying agents for requirements, story writing, coding, documentation, and QA. Coordinated by an orchestrator, their development cycles became standardized and automated—cutting requirements and test design from weeks to days and saving up to 60% of QA time.

3. **Orchestration Patterns** — H2 section documenting the 5 orchestration approaches:
   - **Sequential orchestration**: Agents refine a document or output step by step. Each agent handles one stage and passes results to the next. Best for: Approval workflows, content refinement pipelines.
   - **Concurrent orchestration**: Agents run in parallel and merge results. Best for: Multi-source research, independent subtask execution.
   - **Group chat / maker-checker**: Agents debate and validate outputs together. Best for: High-stakes decisions, compliance review, code review.
   - **Dynamic handoff**: Real-time triage or routing based on context. Best for: IT support, customer service escalation.
   - **Magentic orchestration**: A manager agent coordinates all subtasks until completion. Best for: Complex projects with many dependencies.

4. **Pattern 5: ReAct (Reason + Act)** — H2 section with:
   - **Definition**: The ReAct pattern enables agents to solve problems in real time, especially when static plans fall short. ReAct agents alternate between reasoning and action—taking a step, observing results, and deciding what to do next.
   - **Key capabilities** (bullet list):
     - Adapt to ambiguity and evolving requirements
     - Alternate between reasoning and action iteratively
     - Adjust strategy based on observed results
     - Escalate to humans when scope is exceeded
   - **Citadel placement**: Layer 2 (AI Control Plane) captures step-level traces and reasoning chains. Layer 4 (Security Fabric) monitors each action for policy compliance.
   - **Azure services**: Azure AI Foundry Agent Service, Azure Monitor, Application Insights
   - **Customer example**: In enterprise IT support, a virtual agent powered by ReAct can diagnose issues in real time: it asks clarifying questions, checks system logs, tests possible solutions, and adjusts its strategy as new information becomes available. If the issue grows more complex or falls outside its scope, the agent escalates to a human specialist with a detailed summary.
   - **Implementation guidance**: 2-3 sentences on using Foundry Control Plane traces to observe and debug ReAct reasoning loops.

5. **Pattern Comparison Table** — Include a summary table:

   | Sequential | Step-by-step handoff | Layer 1/2 | APIM + Logic Apps | Approval and refinement pipelines |
   | Concurrent | Parallel execution | Layer 2 | Foundry Agent Service | Multi-source research |
   | ReAct | Adaptive reasoning + action | Layer 2 | Foundry Agent Service | Ambiguous or dynamic problem solving |
