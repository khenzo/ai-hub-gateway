Structure the document as follows:

1. **Overview** — Introduce the three foundational agentic design patterns (tool use, reflection, planning) as building blocks for enterprise automation. Explain that these patterns are designed to be combined and together unlock transformative automation. Mention Microsoft Agent Factory's approach.

2. **Pattern 1: Tool Use** — H2 section with:
   - **Definition**: Modern agents drive real outcomes by interacting directly with enterprise systems—retrieving data, calling APIs, triggering workflows, and executing transactions. Agents surface answers AND complete tasks, update records, and orchestrate workflows end-to-end.
   - **Key capabilities** (bullet list):
     - Invoke APIs and external functions
     - Query databases and knowledge bases
     - Trigger workflows and execute transactions
     - Bridge knowledge and action
   - **Citadel placement**: Layer 1 (Governance Hub / APIM) acts as the unified gateway for all tool invocations, enforcing authentication, rate limiting, and policy. Layer 2 (AI Control Plane) tracks tool usage telemetry.
   - **Azure services**: Azure API Management, API Center, Azure AI Foundry Agent Service tools
   - **Customer example**: Fujitsu transformed its sales proposal process using specialized agents for data analysis, market research, and document creation—each invoking specific APIs and tools. Agents built and assembled entire proposal packages, reducing production time by 67%.
   - **Implementation guidance**: 2-3 sentences on using APIM as the MCP/API gateway for secure tool access.

3. **Pattern 2: Reflection** — H2 section with:
   - **Definition**: Reflection is the ability of agents to assess and improve their own outputs. Agents catch errors and iterate for quality without always depending on humans.
   - **Key capabilities** (bullet list):
     - Self-check outputs for accuracy and completeness
     - Auto-correct missing details or calculations
     - Validate against compliance and messaging standards
     - Reduce errors in high-stakes domains
   - **Citadel placement**: Layer 2 (AI Control Plane) captures evaluation metrics, traces, and quality scores. Layer 4 (Security Fabric / Content Safety) ensures reflected outputs meet policy.
   - **Azure services**: Azure AI Foundry evaluations, Content Safety, Responsible AI dashboard
   - **Customer example**: In compliance and finance, self-checks and review loops help agents auto-correct missing details, double-check calculations, or ensure messages meet standards. GitHub Copilot relies on internal testing and refinement before sharing outputs.
   - **Implementation guidance**: 2-3 sentences on setting up automated evaluation pipelines in Foundry Control Plane.

4. **Pattern 3: Planning** — H2 section with:
   - **Definition**: Planning agents break high-level goals into actionable tasks, track progress, and adapt as requirements shift. Most real business processes are complex journeys with dependencies and branching paths.
   - **Key capabilities** (bullet list):
     - Decompose complex goals into subtasks
     - Track execution progress across multi-step workflows
     - Adapt plans dynamically as conditions change
     - Handle branching logic and dependencies
   - **Citadel placement**: Layer 2 (AI Control Plane) provides execution traces and evaluations for planned flows. Layer 1 (Governance Hub) enforces policy at each step of the plan.
   - **Azure services**: Azure AI Foundry Agent Service, Azure Logic Apps, Azure Functions
   - **Customer example**: ContraForce's Agentic Security Delivery Platform (ASDP) uses planning agents that break down incidents into intake, impact assessment, playbook execution, and escalation. As each phase completes, the agent checks for next steps. Result: 80% of incident investigation and response is now automated and full incident investigation processed for less than $1 per incident.
   - **Implementation guidance**: 2-3 sentences on combining predefined sequences with LLM-generated plans for flexibility.

5. **Pattern Comparison Table** — Include a summary table:
   | Pattern | Core Capability | Citadel Layer | Primary Azure Service | Best For |
   |---------|-----------------|---------------|----------------------|----------|
   | Tool Use | Invoke APIs and execute actions | Layer 1 | APIM + API Center | Bridging knowledge to action |
   | Reflection | Self-assess and improve outputs | Layer 2 | Foundry Evaluations | High-stakes, compliance-heavy domains |
   | Planning | Break goals into actionable steps | Layer 2 | Foundry Agent Service | Complex multi-step workflows |

6. **Combining the Patterns** — Brief paragraph explaining how tool use, reflection, and planning often combine. Planning frequently combines tool use and reflection, showing how these patterns reinforce each other. Plans can be generated dynamically by an LLM or follow a predefined sequence.
