## Phase Context
Phase 2: Core Architecture - Plan 1 of 3
This plan documents the foundational 4-layer architecture model that underpins the Citadel Governance Hub.

## Four-Layer Model Overview
The Foundry Citadel Platform implements a unified, layered approach to AI security and governance:

| Layer | Name | Responsibility | Key Components |
|-------|------|----------------|----------------|
| 🔷 Layer 1 | Governance Hub | Runtime enforcement — unified AI gateway, policy-as-code, identity validation, token rate limiting, content filtering, cost attribution | APIM, API Center, Content Safety |
| 🔶 Layer 2 | AI Control Plane | Observability & compliance — agent traces, AI evaluations, fleet operations, automated compliance checks | Foundry Control Plane, evaluations |
| 🟢 Layer 3 | Agent Identity (Agent 365) | Agent identity & lifecycle — unique agent identities, shadow agent detection, sponsorship model, access packages | Microsoft Entra ID |
| 🛡️ Layer 4 | Security Fabric | Unified protection — Defender threat intelligence, Purview data governance, Entra access control | Defender, Purview, Entra |

The layers are not isolated silos — they form an integrated architecture grounded in the principle of separation of concerns with unified oversight.
</context>
