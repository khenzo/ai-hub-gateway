## Networking Context
The Citadel Governance Hub follows a hub-spoke architecture that integrates with existing Azure Landing Zones:

### Deployment Approaches
1. **Part of the hub network**: Citadel Governance Hub deployed within the existing hub VNet
2. **Part of spoke network**: Citadel deployed in a dedicated spoke VNet with peering to hub

### Network Components
- Central Governance Hub (CGH) - One deployment per environment
- Agent Environment Spokes (CAS) - Multiple deployments per business unit/use case
- VNet peering for hub-spoke connectivity
- Private endpoints for secure service access
- NSGs for subnet-level security
- Azure Firewall for traffic inspection

### Key Networking Principles
- Separation of concerns with unified oversight
- Local freedom in spokes under central governance
- Private connectivity as default
- Defense in depth with layered security
