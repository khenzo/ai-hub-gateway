       Azure Monitor Application Insights + Azure Workbooks, the Agents Playground (test/debug/evaluate),
       and the four built-in agent evaluators. One paragraph, not a deep dive.

    5. **"Five Best Practices" section** — Use `<Steps>` component with detailed sub-content for each step:

       **BP1: Model Selection via Benchmark-Driven Leaderboards**
       - Use Azure AI Foundry leaderboards to compare models on task-specific benchmarks before committing
       - Prevents choosing an under-performing model for the agent's domain
       - Note: model choice affects all downstream evaluation baselines

       **BP2: Continuous Evaluation in Development and Production**
       - Four built-in agent evaluators: Intent Resolution, Task Adherence, Tool Call Accuracy, Response Completeness
       - Quality evaluators: relevance, coherence, fluency
       - Risk/safety evaluators: code vulnerabilities, violence, self-harm, sexual content, hate/unfairness,
         indirect attacks, protected materials
       - Use Agents Playground for interactive debugging before pipeline integration
       - Combine all three evaluator categories (agent, quality, risk) for full coverage

       **BP3: CI/CD Integration**
       - GitHub Actions extension for Azure AI Foundry: run evaluations as PR checks
       - Azure DevOps extension: integrate quality gates into release pipelines
       - Fail builds when evaluation scores drop below threshold
       - Keep evaluation datasets versioned alongside prompt templates

       **BP4: AI Red Teaming Before Production**
       - Azure AI Red Teaming Agent: automated adversarial testing at scale (not just static test sets)
       - PyRIT toolkit: open-source adversarial probing for multi-agent workflows
       - Run before every production promotion — not just at initial launch
       - Use `<Note>` to clarify this is pre-production testing; production monitoring is BP5

       **BP5: Production Monitoring with Tracing, Evaluations, and Alerts**
       - Azure Monitor Application Insights: distributed tracing across agent chains
       - Azure Workbooks: unified dashboard for quality, safety, performance, resource usage metrics
       - Continuous evaluation sampling in production (not just at release time)
       - Microsoft Purview integration: governance evidence for regulatory compliance
       - Credo AI / Saidot: automated mapping to EU AI Act + NIST AI RMF requirements

    6. **"Citadel Layer Mapping for Observability" ASCII diagram**:

       │  Red Teaming (pre-prod)         Layer 2: AI Control Plane                  │
       │  Distributed Tracing            Layer 4: Security Fabric (telemetry)        │
       │  Governance Evidence (Purview)  Layer 1: Governance Hub                     │
       │  Regulatory Mapping (Credo AI)  Layer 1: Governance Hub                     │
