The answer is that Citadel already provides this capability through **Azure API Center** — a unified API catalog that registers, versions, and publishes every API (including MCP servers and A2A endpoints) so that agents and their developers can discover them programmatically or through a searchable portal.

4. **What API Center Provides as an Agent Registry (section heading + prose + bullet list)** — Explain the registry capabilities:
   - **Catalog of all agent-facing APIs**: Every MCP server, A2A endpoint, and traditional REST API that agents can call is registered in API Center with a unique identity
   - **Version and lifecycle tracking**: APIs are versioned (v1, v2, beta) and marked with lifecycle states (design, preview, active, deprecated). Agents can query this to avoid calling deprecated tools.
   - **Owner and governance metadata**: Each registered API has an owner team, a service-line association, and compliance tags. This maps directly to EY's governance tier model.
   - **Developer portal for discovery**: Agent developers can browse the API Center developer portal to find available tools, read specifications, and understand how to authenticate.
   - **Programmatic discovery via API**: The API Center REST API allows agents or orchestrators to query the catalog dynamically — enabling true runtime tool discovery.

5. **From API Registry to Agent Discovery (section heading + prose + table)** — Distinguish the two perspectives:

| Perspective | What They See | How They Interact with API Center |
|---|---|---|
| Platform team | API registration dashboard, governance reports, lifecycle management | Register APIs, assign owners, set versions |
| Agent developer | Searchable developer portal with tool descriptions, schemas, and auth requirements | Browse portal, read OpenAPI specs, copy endpoint URLs |
| Agent runtime | Dynamic catalog query result (JSON) listing available tools and their parameters | Call API Center REST API to discover tools at runtime |

6. **Addressing the 'Future' Requirement Today (section heading + prose)** — This is the core argument of the page. Summarize clearly:
   - EY's framework says agent registration and discovery is future work. Citadel makes it present work.
   - API Center functions as the agent registry. APIM enforces that only registered APIs are reachable. Together, they provide the complete registration-and-discovery pipeline that EY describes.
   - Organizations deploying Citadel do not need to wait for an EY-specific agent discovery service. They can use API Center today and remain aligned with EY's framework direction.
