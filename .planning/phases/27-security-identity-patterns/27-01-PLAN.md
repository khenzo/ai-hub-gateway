## Research Context (from 28-RESEARCH.md)

Key patterns to document:
1. **Zero-Secrets MI Chain**: Agent → APIM (Entra token) → Databricks SQL (MI token) → OneLake (Entra auth)
2. **Governance-by-Construction**: UC column masks applied at query time → masked data flows through APIM → agent receives redacted data
3. **Policy Fragments**: Reusable APIM policy fragments for data governance
4. **Token Exchange**: OBO flow limitations (user principals only)

Key pitfalls to warn against:
- Using secrets (PATs, connection strings) in data path ❌
- OBO flow for service principals ❌
- Complex Python UDFs for masking ❌
- Relying solely on APIM for PII enforcement ❌
- UC limitations with external engines ❌
