2. **The 8 Essential Components** — Document each component in its own H2 section with:
   - Clear definition (2-3 sentences)
   - Key capabilities (bullet list)
   - Citadel placement (which layer(s) and why)
   - Primary Azure/Microsoft service

   The 8 components:
   - **Communication** — A2A protocol, MCP, message routing. Citadel Layer 1 (APIM gateway enforces authentication, rate limiting, and policy on all agent communication). Key service: Azure API Management.
   - **Discovery** — Agent registries, capability directories, semantic discovery. Citadel Layer 2/3 (AI Control Plane tracks capabilities, Agent 365 maintains identity registry). Key services: Foundry Control Plane, Agent 365.
   - **Identity** — Unique agent identities, sponsorship, lifecycle, access packages. Citadel Layer 3 (Agent Identity). Key service: Microsoft Entra ID / Agent 365.
   - **Tools** — MCP servers, APIs, functions, external system connectors. Citadel Layer 1 (Governance Hub acts as unified gateway for all tool invocations). Key service: Azure API Management + API Center.
   - **Orchestration** — Multi-agent workflows, planning, delegation, state management. Citadel Layer 2 (AI Control Plane provides execution traces and evaluations for orchestrated flows). Key service: Azure AI Foundry + Agent Framework.
   - **Observability** — Execution traces, evaluations, telemetry, compliance dashboards. Citadel Layer 2 (Foundry Control Plane captures agent behavior and quality metrics). Key service: Foundry Control Plane + Azure Monitor.
   - **Memory** — Knowledge bases, RAG, conversation state, long-term memory. Maps to Foundry IQ within the intelligence layer; Citadel Layer 2 evaluates retrieval quality and Layer 4 governs data access. Key service: Azure AI Foundry (knowledge bases) + Azure AI Search.
   - **Governance** — Policies, compliance, content safety, PII protection, audit. Spans all 4 Citadel layers (runtime enforcement, observability, identity governance, security fabric). Key services: Azure Policy, Content Safety, Purview, Defender.

3. **Component-to-Citadel Mapping Table** — Include a summary table:
   | Component | Citadel Layer | Primary Service | Purpose |

   | Discovery | Layer 2/3 | Foundry / Agent 365 | Capability and identity registry |
   | Identity | Layer 3 | Entra ID / Agent 365 | Agent lifecycle and access |
   | Tools | Layer 1 | APIM / API Center | Unified tool gateway |
   | Orchestration | Layer 2 | Azure AI Foundry | Multi-agent workflow management |
   | Observability | Layer 2 | Foundry Control Plane | Traces, evaluations, telemetry |
   | Memory | Layer 2/4 | Foundry / AI Search | Knowledge retrieval and state |
   | Governance | All Layers | Policy / Defender / Purview | Compliance and security |

4. **Integration with Intelligence Layers** — Brief paragraph explaining how the stack components consume Work IQ, Fabric IQ, and Foundry IQ.
