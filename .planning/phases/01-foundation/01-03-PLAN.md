<task type="auto">
  <name>Create documentation style guide</name>
  <files>STYLE_GUIDE.md</files>
  <action>
    Create comprehensive style guide covering:
    
    Voice and Tone:
    - Professional but approachable
    - Use "you" for direct address
    - Active voice preferred
    - Clear, concise language
    
    Structure:
    - Page frontmatter requirements (title, description, icon)
    - Heading hierarchy (H1 once, logical progression)
    - Paragraph length (3-5 sentences max)
    - List formatting (parallel structure)
    
    Technical Writing:
    - Code blocks with language tags
    - Use Callouts for warnings, tips, info
    - Use Cards for related links
    - Use Tabs for alternate approaches
    - Use Steps for procedures
    
    Mintlify Components:
    - Available components and when to use
    - Props and configuration
    - Examples for each component
    
    Formatting:
    - Use backticks for code, commands, file paths
    - Bold for UI elements, emphasis
    - Italics sparingly
    - Links: descriptive text, not "click here"
    
    Images and Diagrams:
    - Use SVG when possible
    - Alt text for accessibility
    - Consistent naming convention
    - Place in images/ directory
    
    Terminology:
    - Consistent use of product names
    - Acronyms spelled out first use
    - Glossary for domain terms
  </action>
  <verify>STYLE_GUIDE.md covers all major aspects with examples</verify>
  <done>Style guide created with comprehensive standards</done>
</task>
