   - 3 Build image — none — n/a

   - 6 Push to ACR — Layer 4 (Security Fabric) — ACR pulls policy tag from Defender scan; unsigned images rejected
   - 7 Helm upgrade — Layer 1 (Governance Hub) — Namespace label `citadel.io/managed=true` enforces network policy; APIM backend URL injected via Helm values
   - 8 Apply APIM policy — Layer 1 (Governance Hub) — `az apim api policy create` applies inbound.xml (subscription key + JWT) and outbound.xml (response scrubbing) from `/policies/` directory
   - 9 Register agent trace — Layer 2 (AI Control Plane) — `az ml online-endpoint` or Foundry SDK call registers the deployment with the AI Control Plane for trace collection

   - Environment variables at top: IMAGE_TAG, NAMESPACE (citadel-agents), HELM_RELEASE
   - Job `cd`: runs-on ubuntu-latest, steps:
     - Checkout

   - Add inline YAML comments on each step for Citadel layer
