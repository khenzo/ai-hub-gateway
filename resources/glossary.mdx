# Glossary

Definitions of key terms and concepts used throughout the AI Landing Zone documentation.

## A

**Access Contract**  
A policy document defining the terms under which an application can consume AI services, including rate limits, cost budgets, content filters, and data residency requirements.

**AI Gateway**  
The Citadel component built on Azure API Management that serves as the unified entry point for all AI service requests, handling authentication, rate limiting, and request routing.

**AI Landing Zone**  
A pre-configured, secure environment for deploying AI workloads, including network isolation, identity management, monitoring infrastructure, and AI service integration points.

**AI Registry**  
The Citadel component that maintains a catalog of available AI services, their configurations, and associated access contracts.

**APIM (API Management)**  
Azure API Management service used as the foundation for the Citadel Gateway to manage, secure, and observe API calls to AI services.

## C

**Citadel**  
The governance hub that provides centralized control and visibility over AI infrastructure deployed across an organization. Implements the 4-layer architecture.

**Citadel Gateway**  
See "AI Gateway".

**Content Filter**  
Safety mechanisms applied to AI service inputs and outputs to prevent harmful, inappropriate, or policy-violating content.

**Core Components**  
The four primary Citadel components: Gateway, Registry, Observability Platform, and Policy Engine.

## D

**Defense in Depth**  
A security strategy using multiple layers of security controls to protect AI infrastructure.

## F

**4-Layer Model**  
The Citadel architecture organization: Layer 1 (Foundation), Layer 2 (AI Services), Layer 3 (Governance), Layer 4 (Applications).

**Foundation Layer**  
Layer 1 of the architecture providing core infrastructure including networking, identity, security services, and monitoring foundation.

## G

**Governance Layer**  
Layer 3 of the architecture responsible for policy enforcement, access contracts, rate limiting, and compliance controls.

## H

**Hub and Spoke**  
A network topology where the Citadel hub contains shared services and landing zone spokes connect to the hub for shared services access.

## I

**Input Token**  
A unit of billing based on text sent to an AI model. Part of token-based pricing for AI services.

## L

**Landing Zone**  
A spoke network containing AI services and applications, connected to the Citadel hub for governance and shared services.

**Layer 1 (Foundation)**  
Core infrastructure layer: Virtual Networks, Azure AD, Key Vault, Log Analytics.

**Layer 2 (AI Services)**  
Managed AI capabilities: Azure OpenAI, AI Search, ML Workspace.

**Layer 3 (Governance)**  
Policy and control layer: Access Contracts, rate limiting, content filtering.

**Layer 4 (Applications)**  
AI-powered solutions: Line-of-business apps, chatbots, custom integrations.

**Least Privilege**  
A security principle granting only the minimum necessary permissions to identities and services.

## O

**Observability Platform**  
The Citadel component providing monitoring, logging, metrics, and analytics across all AI infrastructure.

**Output Token**  
A unit of billing based on text generated by an AI model. Part of token-based pricing.

## P

**Policy Engine**  
The Citadel component that enforces governance policies and access contracts across AI service consumption.

**Private Endpoint**  
A network interface connecting Azure services privately to Virtual Networks, eliminating public internet exposure.

## R

**Rate Limiting**  
Controls on the number of requests or tokens an application can consume within a time period.

**RBAC (Role-Based Access Control)**  
Azure's access management system controlling what actions users and services can perform on resources.

## S

**Service Registry**  
See "AI Registry".

**Spoke**  
A Virtual Network containing AI workload resources that connects to the Citadel hub via peering.

## T

**Token**  
The billing unit for AI model consumption, typically representing a word or part of a word. Input and output tokens are counted separately.

**Token-Based Pricing**  
The billing model for AI services based on the number of tokens processed (input + output).

## Z

**Zero Trust**  
A security model requiring explicit verification for every access request, regardless of origin, following "never trust, always verify."
