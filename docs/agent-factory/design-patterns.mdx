# Agentic Design Patterns

The three foundational agentic design patterns—**tool use**, **reflection**, and **planning**—are the building blocks for enterprise automation. They are designed to be combined, and together they unlock transformative automation that bridges knowledge and action.

Microsoft Agent Factory provides the platforms, intelligence layers, and governance model to put these patterns into production. When deployed on Citadel, each pattern gains enterprise-grade security, observability, and policy enforcement.

## Pattern 1: Tool Use

Modern agents drive real outcomes by interacting directly with enterprise systems—retrieving data, calling APIs, triggering workflows, and executing transactions. Agents surface answers **and** complete tasks, update records, and orchestrate workflows end-to-end.

### Key Capabilities

- Invoke APIs and external functions
- Query databases and knowledge bases
- Trigger workflows and execute transactions
- Bridge knowledge and action

### Citadel Placement

**Layer 1 (Governance Hub / APIM)** acts as the unified gateway for all tool invocations, enforcing authentication, rate limiting, and policy. **Layer 2 (AI Control Plane)** tracks tool usage telemetry through the Foundry Control Plane.

### Azure Services

- Azure API Management
- API Center
- Azure AI Foundry Agent Service tools

### Customer Example

**Fujitsu** transformed its sales proposal process using specialized agents for data analysis, market research, and document creation—each invoking specific APIs and tools. Instead of simply answering "what should we pitch," agents built and assembled entire proposal packages, **reducing production time by 67%**.

### Implementation Guidance

Use Azure API Management as the MCP and API gateway for all agent tool access. This centralizes authentication, enables rate limiting, and provides a single enforcement point for enterprise policies. Route all Foundry Agent Service and Copilot Studio API calls through APIM to ensure consistent governance.

## Pattern 2: Reflection

Reflection is the ability of agents to assess and improve their own outputs. Agents catch errors and iterate for quality without always depending on humans. In high-stakes domains, a single error can be costly—self-improving loops reduce risk and build enterprise confidence.

### Key Capabilities

- Self-check outputs for accuracy and completeness
- Auto-correct missing details or calculations
- Validate against compliance and messaging standards
- Reduce errors in high-stakes domains

### Citadel Placement

**Layer 2 (AI Control Plane)** captures evaluation metrics, traces, and quality scores. **Layer 4 (Security Fabric / Content Safety)** ensures reflected outputs meet policy before they reach users or downstream systems.

### Azure Services

- Azure AI Foundry evaluations
- Azure AI Content Safety
- Responsible AI dashboard

### Customer Example

In compliance and finance, self-checks and review loops help agents auto-correct missing details, double-check calculations, or ensure messages meet standards. **GitHub Copilot** relies on internal testing and refinement before sharing outputs, demonstrating how reflection improves reliability at scale.

### Implementation Guidance

Set up automated evaluation pipelines in the Azure AI Foundry Control Plane to run on agent outputs. Define success criteria for accuracy, completeness, and tone. Use Content Safety as a gate in Layer 4 to block or flag outputs that fail policy checks before they are delivered.

## Pattern 3: Planning

Planning agents break high-level goals into actionable tasks, track progress, and adapt as requirements shift. Most real business processes are complex journeys with dependencies and branching paths—not single-step actions.

### Key Capabilities

- Decompose complex goals into subtasks
- Track execution progress across multi-step workflows
- Adapt plans dynamically as conditions change
- Handle branching logic and dependencies

### Citadel Placement

**Layer 2 (AI Control Plane)** provides execution traces and evaluations for planned flows, enabling observability into every step. **Layer 1 (Governance Hub)** enforces policy at each step of the plan, ensuring no action bypasses enterprise controls.

### Azure Services

- Azure AI Foundry Agent Service
- Azure Logic Apps
- Azure Functions

### Customer Example

**ContraForce's Agentic Security Delivery Platform (ASDP)** uses planning agents that break down incidents into intake, impact assessment, playbook execution, and escalation. As each phase completes, the agent checks for next steps. The result: **80% of incident investigation and response is now automated**, with full incident investigation processed for **less than $1 per incident**.

### Implementation Guidance

Combine predefined sequences with LLM-generated plans for flexibility. Use Azure Logic Apps for deterministic workflow steps and Azure AI Foundry Agent Service for dynamic planning. Enable step-level tracing in the Foundry Control Plane so every plan execution is observable and auditable.

## Pattern Comparison

| Pattern | Core Capability | Citadel Layer | Primary Azure Service | Best For |
|---------|-----------------|---------------|----------------------|----------|
| Tool Use | Invoke APIs and execute actions | Layer 1 | APIM + API Center | Bridging knowledge to action |
| Reflection | Self-assess and improve outputs | Layer 2 | Foundry Evaluations | High-stakes, compliance-heavy domains |
| Planning | Break goals into actionable steps | Layer 2 | Foundry Agent Service | Complex multi-step workflows |

## Combining the Patterns

Tool use, reflection, and planning rarely operate in isolation. In practice, they reinforce each other:

- A **planning agent** decomposes a goal, then invokes tools to execute each step
- A **tool-use agent** reflects on API responses before deciding the next action
- A **reflection loop** validates the output of a planned workflow before it proceeds

Planning frequently combines tool use and reflection, showing how these patterns reinforce each other. Plans can be generated dynamically by an LLM or follow a predefined sequence, whichever fits the enterprise need.
