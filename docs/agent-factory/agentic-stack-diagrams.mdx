# Agentic Web Stack Diagrams

These diagrams visualize how the 8 agentic web stack components map to the Citadel 4-layer architecture and the AI Landing Zone infrastructure. They are designed to help architects understand component placement, layer interactions, and how the agentic web stack integrates with the existing Citadel governance model.

## Overview

The agentic web stack defines 8 essential capabilities for enterprise agent systems:

1. **Communication** — A2A/MCP protocol layer
2. **Discovery** — Agent and tool registries
3. **Identity** — Agent lifecycle and access
4. **Tools** — MCP servers and API gateway
5. **Orchestration** — Multi-agent workflow coordination
6. **Observability** — Traces, evaluations, and telemetry
7. **Memory** — Knowledge bases and state
8. **Governance** — Policy, compliance, and security

Each component maps to one or more Citadel layers, creating a unified governance envelope around every agent interaction.

## Diagram 2: Component-to-Layer Mapping (Detailed)

```
┌─────────────────────────────────────────────────────────────────────────┐
│           COMPONENT-TO-LAYER MAPPING                                   │
│                                                                         │
│   Communication  ═══════════════════════►  Layer 1 (APIM Gateway)      │
│   Tools          ═══════════════════════►  Layer 1 (API Center)        │
│   Identity       ═══════════════════════►  Layer 3 (Agent 365)         │
│   Discovery      ═══════════════════════►  Layer 2/3 (Foundry + 365)   │
│   Orchestration  ═══════════════════════►  Layer 2 (Foundry Control)   │
│   Observability  ═══════════════════════►  Layer 2 (Foundry Control)   │
│   Memory         ═══════════════════════►  Layer 2/4 (Foundry + Purv)  │
│   Governance     ═══════════════════════►  All Layers (Policy + Def)   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**What this illustrates:** This mapping shows the primary layer home for each stack component. Communication and Tools anchor in Layer 1 because they require runtime gateway enforcement. Identity and Discovery span Layer 2/3 because they combine capability registries with authoritative identity. Orchestration and Observability live in Layer 2 where the Foundry Control Plane manages execution and telemetry. Memory bridges Layer 2 (knowledge retrieval) and Layer 4 (data governance). Governance cuts across all layers as the cross-cutting concern.

                            │
                            ▼
              ┌─────────────────────────────┐
              │         Layer 2             │

**What this illustrates:** This flow diagram traces a single agent request as it descends through the stack. Communication initiates the request, which is then filtered by Layer 1 runtime policies. Layer 3 validates the agent's identity before Layer 2 orchestrates execution and captures observability. Layer 4 provides continuous governance and security monitoring throughout the entire journey.
