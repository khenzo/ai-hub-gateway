# Agent Factory on Citadel

The **Enterprise Agent Factory** and the **Citadel 4-Layer Architecture** are complementary frameworks. The Agent Factory provides the intelligence layers, builder toolchains, and operating model for creating enterprise AI agents. Citadel provides the governance, security, and observability foundation required to deploy those agents at scale — independent of which frameworks or models are used.

## Mapping Overview

Citadel is designed to be the **governance and security foundation** for any agentic deployment. Together, they enable organizations to:

- Build agents with the right tools for every persona and framework
- Ground them in enterprise intelligence (Workspace, Business Data, Enterprise Knowledge)
- Deploy them with consistent governance, identity, and security
- Observe and manage them as first-class enterprise workloads

## Citadel 4-Layer + Agent Factory Mapping

| Citadel Layer | Agent Factory Component | Integration Point |
|---------------|------------------------|-------------------|
| **Layer 1: Governance Hub** | APIM Gateway, API Center | All agent API traffic, MCP server calls, and tool invocations route through CGH |
| **Layer 2: AI Control Plane** | Foundry Control Plane, evaluations | Agent observability, execution traces, and quality evaluations |
| **Layer 3: Agent Identity** | Identity & lifecycle registry | Unique agent identities, sponsorship models, access packages |
| **Layer 4: Security Fabric** | Defender, Purview, Entra | Protects agent workloads and governs access to all intelligence sources |

Each Citadel layer maps directly to a critical component of the Agent Factory operating model, ensuring that agent innovation does not outpace governance.

## Intelligence Layers in the Landing Zone

### Workspace Intelligence → Layer 1 + Layer 4

Workspace Intelligence draws on signals from collaboration systems (email, calendar, messaging). When agents access this employee context:

- **Layer 1 (Governance Hub)** enforces runtime policies via APIM on all workspace API calls
- **Layer 4 (Security Fabric)** governs data access and validates permissions before any workspace data reaches an agent

### Business Data Intelligence → Layer 2 Observability

Business Data Intelligence provides operational context from data warehouses, BI platforms, and databases:

- **Layer 2 (AI Control Plane)** captures data access telemetry through the control plane
- Evaluations monitor what intelligence agents consume and how they reason over it
- Compliance dashboards track data access patterns across the agent fleet

### Enterprise Knowledge Intelligence → Layer 1 + Layer 2

Enterprise Knowledge Intelligence connects institutional knowledge via MCP servers, internal docs, and the web:

- **Layer 1 (Governance Hub)** acts as the gateway for all MCP server traffic, enforcing authentication, rate limiting, and content filtering
- **Layer 2 (AI Control Plane)** evaluates knowledge retrieval quality and monitors access patterns
- Permission-respecting knowledge bases integrate with **Layer 3 (Agent Identity)** for user-scoped access

## Builder Toolchains & Deployment Patterns

### Low-Code / No-Code Agents

**Deployment pattern:** Lightweight, integration-friendly, governed via Layer 1

- Route all agent API calls through the Citadel Governance Hub (APIM)
- Use Layer 3 for agent identity and lifecycle management
- Apply Layer 4 content filtering and data governance policies

**Best for:** Power users building self-service, HR/IT support, and sales enablement agents

### Framework-Based Agents (LangChain, LangGraph, Claude Agent SDK, OpenAI Agents SDK, Microsoft Agent Framework)

**Deployment pattern:** Enterprise-scale, custom orchestration, full 4-layer governance

- Deploy agent runtimes in a Citadel spoke with VNet peering to the CGH
- Route all model inference through APIM for runtime policy enforcement
- Use the control plane (Layer 2) for end-to-end observability and evaluations
- Integrate with Layer 3 for identity, sponsorship, and access packages
- Use APIM as the MCP gateway for all tool invocations

**Best for:** Developers building complex, multi-step agents with custom integrations and open frameworks

## Agentic Web Stack on Citadel

The agentic web stack defines 8 essential capabilities that enterprise agent systems require. Each maps to a Citadel layer:

- **Communication** → Layer 1 (APIM)
- **Discovery** → Layer 2/3 (Control Plane / Identity Registry)
- **Identity** → Layer 3 (Agent Identity)
- **Tools** → Layer 1 (APIM / API Center)
- **Orchestration** → Layer 2 (AI Control Plane)
- **Observability** → Layer 2 (AI Control Plane)
- **Memory** → Enterprise Knowledge layer / Layer 2
- **Governance** → All 4 layers

## Deployment Recommendations

1. **Route all agent traffic through Layer 1 (APIM)**
   - All framework-based and low-code agent API calls should pass through the CGH gateway
   - Enables unified rate limiting, cost attribution, and policy enforcement across all toolchains

2. **Use Layer 2 for agent observability**
   - Enable execution traces for all agent runs
   - Configure evaluations to monitor output quality and safety

3. **Register every agent with Layer 3 (Agent Identity)**
   - Unique identity per agent, regardless of framework
   - Implement sponsorship models and lifecycle management
   - Use access packages for scoped permissions

4. **Use Layer 4 for threat detection and data governance**
   - Apply content safety policies to all agent inputs and outputs
   - Govern access to intelligence sources (workspace, business data, enterprise knowledge)
   - Enforce PII protection across all data flows
