# Agent Factory on Citadel

The **Microsoft Agent Factory** and the **Citadel 4-Layer Architecture** are complementary frameworks. Agent Factory provides the intelligence layers, builder platforms, and operating model for creating enterprise AI agents. Citadel provides the governance, security, and observability foundation required to deploy those agents at scale.

## Mapping Overview

Citadel is designed to be the **governance and security foundation** for Microsoft Agent Factory deployments. Together, they enable organizations to:

- Build agents with the right tools for every persona
- Ground them in enterprise intelligence (Work IQ, Fabric IQ, Foundry IQ)
- Deploy them with consistent governance, identity, and security
- Observe and manage them as first-class enterprise workloads

## Citadel 4-Layer + Agent Factory Mapping

| Citadel Layer | Agent Factory Component | Integration Point |
|---------------|------------------------|-------------------|
| **Layer 1: Governance Hub** | APIM Gateway, API Center | Foundry + Copilot Studio agents route through CGH |
| **Layer 2: AI Control Plane** | Foundry Control Plane, evaluations | Agent Factory observability and compliance brain |
| **Layer 3: Agent Identity** | Agent 365 | Unified agent identity, sponsorship, lifecycle |
| **Layer 4: Security Fabric** | Defender, Purview, Entra | Protects agent workloads across all IQ layers |

Each Citadel layer maps directly to a critical component of the Agent Factory operating model, ensuring that agent innovation does not outpace governance.

## Intelligence Layers in the Landing Zone

### Work IQ → Layer 1 Gateway Enforcement

Work IQ draws on Microsoft 365 signals (emails, calendar, Teams). When agents access this employee context:

- **Layer 1 (Governance Hub)** enforces runtime policies via APIM
- **Layer 4 (Security Fabric)** ensures Purview governs data access and Entra validates permissions
- All Work IQ agent traffic routes through the CGH for unified policy enforcement

### Fabric IQ → Layer 2 Observability

Fabric IQ provides business data context from Power BI, Dataverse, and Azure SQL:

- **Layer 2 (AI Control Plane)** captures data access telemetry through the Foundry Control Plane
- Evaluations monitor what intelligence agents consume and how they reason over it
- Compliance dashboards track Fabric IQ access patterns across the agent fleet

### Foundry IQ → Layer 1 + Layer 2

Foundry IQ connects enterprise knowledge via MCP servers, internal data, and the public web:

- **Layer 1 (Governance Hub)** acts as the gateway for all MCP server traffic, enforcing authentication, rate limiting, and content filtering
- **Layer 2 (AI Control Plane)** evaluates knowledge retrieval quality and monitors knowledge base access
- Permission-respecting knowledge bases integrate with Layer 3 (Agent Identity) for user-scoped access

## Builder Platforms & Deployment Patterns

### Copilot Studio Agents

**Deployment pattern:** Lightweight, M365-integrated, governed via Layer 1

- Route all Copilot Studio agent API calls through the Citadel Governance Hub (APIM)
- Use Layer 3 (Agent 365) for agent identity and lifecycle management
- Apply Layer 4 content filtering and data governance policies

**Best for:** Power users building self-service, HR/IT support, and sales enablement agents

### Foundry + Agent Framework

**Deployment pattern:** Enterprise-scale, custom orchestration, full 4-layer governance

- Deploy Foundry Hub in a Citadel spoke with VNet peering to the CGH
- Route all model inference through APIM for runtime policy enforcement
- Use the Foundry Control Plane (Layer 2) for end-to-end observability and evaluations
- Integrate with Agent 365 (Layer 3) for identity, sponsorship, and access packages

**Best for:** Developers building complex, multi-step agents with custom integrations

## Agentic Web Stack on Citadel

The agentic web stack defines the 8 essential capabilities that enterprise agent systems require. Each capability maps directly to a Citadel layer.

- **Communication** → Layer 1 (APIM)
- **Discovery** → Layer 2/3 (Foundry / Agent 365)
- **Identity** → Layer 3 (Agent 365)
- **Tools** → Layer 1 (APIM / API Center)
- **Orchestration** → Layer 2 (Foundry Control Plane)
- **Observability** → Layer 2 (Foundry Control Plane)
- **Memory** → Foundry IQ / Layer 2/4
- **Governance** → All 4 layers

## Deployment Recommendations

1. **Use Citadel Layer 1 to route all Agent Factory agent traffic**
   - All Copilot Studio and Foundry agent API calls should pass through the CGH APIM gateway
   - This enables unified rate limiting, cost attribution, and policy enforcement

2. **Use Layer 2 for Foundry Control Plane observability on agent outputs**
   - Enable execution traces and AI evaluations for all Foundry-hosted agents
   - Configure compliance dashboards to monitor Agent Factory workloads

3. **Use Layer 3 (Agent 365) for identity and lifecycle of all agents**
   - Register every agent with a unique identity
   - Implement sponsorship models and access packages for agent permissions
   - Use lifecycle management for versioning and retirement

4. **Use Layer 4 for threat detection and data governance across IQ sources**
   - Deploy Defender for threat intelligence on agent workloads
   - Use Purview to govern access to Work IQ, Fabric IQ, and Foundry IQ data sources
   - Apply Entra conditional access policies to agent service principals
