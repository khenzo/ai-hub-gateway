# Data and Analytics Agents

The **Data and Analytics Agents** use case lets employees ask questions in natural language and get answers grounded in actual business data. Instead of hunting for the right report or waiting for an analyst to pull numbers, users interact with agents that understand the business context and deliver insights on demand.

## The Challenge

Dashboards exist everywhere, but employees still ask analysts to pull numbers. People know data exists but struggle to find the right report or interpret what they see. Analysts spend hours each week answering ad-hoc questions instead of doing strategic analysis.

The consequences compound over time:

- **Decision delays** while employees wait for answers that should be self-service
- **Analyst burnout** from repetitive data requests that don't require deep expertise
- **Underutilized data investments** — organizations have spent heavily on BI tools, but adoption remains siloed
- **Inconsistent answers** when the same question is answered differently depending on who is asked

For many organizations, the bottleneck isn't data availability. It's the friction between business questions and the technical steps required to answer them.

## The Opportunity

Let employees ask questions in natural language and get answers grounded in your actual business data. *"What were Q3 sales by region?"* returns an answer drawn directly from your semantic layer. Complex queries can be asked, answered, and included in executive reports without opening a BI tool.

Analysts reclaim time for high-value work—modeling, forecasting, and strategic recommendations—while the organization gets faster, more consistent access to insights.

The business outcomes include:

- **Faster decision-making** with answers available in seconds, not hours or days
- **Analyst capacity unlocked** for work that drives competitive advantage
- **Higher data literacy** as employees engage with data through conversation
- **Improved data trust** when answers come from a single, governed semantic layer

## Builder Personas & Approaches

| Persona | Approach | Tool |
|---------|----------|------|
| **Business User** | Use Copilot's native Power BI integration to ask natural language questions over existing datasets. No building required. | M365 Copilot + Fabric IQ |
| **Data Engineer** | Build Fabric Data Agents that expose curated datasets for natural language queries. Define the semantic layer—business terms, relationships, and metrics. Configure scheduled reports, threshold-based alerts, and data quality checks. | Fabric Data Agent + Fabric IQ |
| **Developer** | Create sophisticated analytics agents that expose APIs for embedding in AI applications and customer-facing solutions. Integrate non-standard data sources and custom orchestration. | Foundry + Agent Framework |

## IQ Layer Mapping

Data and analytics agents rely on the intelligence layers to bridge business questions and technical data sources:

- **Fabric IQ** provides the semantic layer for business data, metrics, and relationships. It ensures that when an employee asks about "Q3 sales," the agent understands the official definition, the right tables, and the correct filters.
- **Foundry IQ** adds unstructured knowledge and report context—documentation, definitions, and institutional knowledge that help the agent explain *why* a number is what it is.
- **Work IQ** enables sharing insights with the right stakeholders, routing reports, and surfacing data in the context of an employee's actual work.

## Citadel Placement

Deploying data and analytics agents on Citadel ensures data access is governed, observable, and compliant:

- **Layer 2 (AI Control Plane)** captures data access telemetry and evaluation metrics through the Foundry Control Plane. Monitor what intelligence agents consume and how they reason over it.
- **Layer 4 (Security Fabric / Purview)** governs data access and ensures compliance. Use Purview to enforce classification, sensitivity labels, and access policies across all data sources the agent queries.
- **Layer 1 (Governance Hub / APIM)** provides API access to non-standard data sources. Route all agent API calls through APIM for unified authentication, rate limiting, and policy enforcement.

## Customer Example

A **large banking customer** with mature Power BI deployments uses this pattern to unlock their existing data investment. After a training program for their US teams, the breakthrough moment came when business leaders realized they could ask questions about their own divisional data and get accurate, grounded answers.

When employees see an agent correctly report last quarter's KPIs directly into research documents, skepticism turns to adoption. The data engineering team maintains the semantic layer, while business users interact without writing a query.

## Buying Pattern Fit

- **Primary audience:** Organizations with moderate to heavy Power BI usage and an established analytics function
- **Key indicators:** Analytics team demand pressure, backlog of ad-hoc reporting requests, or underutilized BI investments
- **Quick win:** Enable natural language over an existing, well-governed Power BI dataset and measure analyst time saved within 60 days
