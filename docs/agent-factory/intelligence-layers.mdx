# Intelligence Layers

Enterprise agents are only as useful as the context they can access. The Agent Factory is built on three interconnected intelligence layers that provide agents with the grounding they need to operate effectively and responsibly — independent of any specific platform or vendor.

## Workspace Intelligence

**Workspace Intelligence understands how people work.** It draws on signals from collaboration systems — email, calendar, messaging, documents, and task management — to build a rich understanding of each employee's work context.

### Primary Sources

| Source | Context Provided |
|--------|------------------|
| **Email & Calendar** | Meetings, priorities, commitments, travel |
| **Messaging & Chat** | Conversations, project channels, meeting transcripts |
| **Document Systems** | Documents authored, shared, and accessed |
| **Task & Project Tools** | Tasks, deadlines, dependencies, assignments |

### Agent Value

When an employee asks an agent *"what should I prioritize today?"*, Workspace Intelligence provides context based on their actual work — not generic productivity advice. It enables:

- Personal prioritization and task routing
- Intelligent meeting preparation
- Context-aware responses in the flow of work
- Delegation path awareness

**Citadel placement:** **Layer 4 (Security Fabric)** governs all access to personal workspace data. **Layer 1 (Governance Hub)** enforces runtime policies on workspace API calls via APIM, ensuring that agents only access data within their authorized scope.

## Business Data Intelligence

**Business Data Intelligence understands how the business operates.** It provides a unified view of business data from BI platforms, operational databases, and data warehouses — understanding your business terms, processes, and what steps agents can take to improve operations.

### Primary Sources

| Source | Context Provided |
|--------|------------------|
| **BI Platforms** | Metrics, KPIs, dashboards, reports |
| **CRM / ERP Systems** | Business entities, relationships, workflows |
| **Operational Databases** | Transactional history, application state |
| **Data Warehouses** | Semantic layer, business terms, aggregate metrics |

### Agent Value

Business Data Intelligence acts as the **semantic layer** for agent operations. It enables agents to:

- Make decisions grounded in real operational metrics
- Understand business terminology and relationships
- Query data accurately using natural language
- Generate reports and alerts autonomously

**Citadel placement:** **Layer 2 (AI Control Plane)** captures data access telemetry and monitors what intelligence agents consume. Evaluations track how agents reason over business data, while compliance dashboards surface access patterns across the agent fleet.

## Enterprise Knowledge Intelligence

**Enterprise Knowledge Intelligence connects all institutional knowledge.** It serves as a managed knowledge system that grounds agents in structured and unstructured content — SOPs, policies, manuals, technical documentation — as well as external sources via Model Context Protocol (MCP) and the public web.

### Primary Sources

| Source | Context Provided |
|--------|------------------|
| **Internal Documentation** | SOPs, policies, procedures, manuals |
| **MCP-Connected Systems** | APIs, databases, enterprise systems via MCP servers |
| **Vector Knowledge Bases** | RAG-indexed unstructured content for semantic retrieval |
| **Public Web** | External research, news, competitive intelligence |

### Agent Value

Enterprise Knowledge Intelligence delivers the **knowledge foundation** for agents:

- **RAG grounding**: Reusable knowledge bases for retrieval-augmented generation across all sources
- **Permission-respecting access**: Knowledge retrieval honors existing user permissions
- **MCP connectivity**: Unified protocol for connecting agents to any enterprise system
- **Enterprise-wide coverage**: SOPs, manuals, policies, and institutional knowledge

**Citadel placement:** **Layer 1 (Governance Hub)** acts as the gateway for all MCP server traffic, enforcing authentication, rate limiting, and content filtering. **Layer 2** evaluates knowledge retrieval quality and monitors knowledge base access patterns.

## How the Layers Work Together

| Layer | Context Type | Primary Sources | Agent Value |
|-------|--------------|-----------------|-------------|
| **Workspace Intelligence** | People context | Collaboration tools | Personal prioritization, routing |
| **Business Data Intelligence** | Operational context | BI, databases, ERP | Decisions grounded in real metrics |
| **Enterprise Knowledge Intelligence** | Institutional knowledge | Docs + MCP + web | Grounded answers, SOPs, manuals |

The intelligence layers do not operate in isolation. They feed into the agent harness and context engineering layers that control what the model sees and how it acts.

## Industry Example

For a large industrial equipment manufacturer, the challenge isn't data — it's knowing what matters right now across plants, shifts, and systems.

**Scenario:** A line operator asks an agent, *"What's causing this deviation at station 23, and what should I do?"*

1. **Workspace Intelligence** knows the operator and their shift history
2. **Business Data Intelligence** correlates machines and line operations
3. **Enterprise Knowledge Intelligence** retrieves the SOP and maintenance manuals

**Result:** The agent provides clear answers about the missed maintenance procedure and resolution steps — reducing downtime and accelerating decisions.
