# Builder Personas

The Enterprise Agent Factory supports the full spectrum of builders so you can activate talent wherever it exists in your organization. The key is matching the right people to the right problems with the right tools — regardless of which AI frameworks or platforms your organization uses.

## Builder Personas Overview

| Persona | Description | Primary Toolchain |
|---------|-------------|-------------------|
| **Business User** | Consumes insights and interacts with agents without building | Chat interfaces, embedded agents |
| **Power User / Maker** | Builds solutions without writing code; uses visual/low-code tools | Low-code agent builders, process automation platforms |
| **Developer** | Builds custom agents with full control over architecture and orchestration | LangChain, LangGraph, Claude Agent SDK, OpenAI Agents SDK, custom frameworks |

Most successful organizations activate **all three personas**. Makers create self-service agents with governed tools, while developers build platforms that scale.

## Scenario 1: Employee Self-Service (HR and IT Support)

### Challenge

Help desks are overwhelmed with routine questions. Employees can't easily find answers in policies and end up calling IT or HR for everything. Knowledge is scattered across documentation, wikis, and institutional memory.

### Opportunity

Deploy an agent that answers routine questions instantly, freeing help desk staff to focus on complex issues. Employees get answers in the flow of work through the channels they already use. Tickets are created automatically when the user doesn't find what they need.

### Who Should Act

If your ops team has business process knowledge, they can own the agent using a low-code builder end-to-end. If the solution needs deep integration with custom systems, strict network isolation, or bespoke UX, developers should lead — leveraging the same intelligence layers.

### Builder Approaches

| Persona | Approach | Toolchain |
|---------|----------|-----------|
| **Power User** | Build a self-service process agent using a low-code platform, grounding it in enterprise knowledge via RAG over policy documents. Configure connectors to ITSM and HR systems. | Low-code agent builder + Enterprise Knowledge layer |
| **Developer** | Build an agent with full control over conversation logic using LangChain/LangGraph. Expose internal systems as MCP servers through the APIM gateway. Deploy with Citadel governance. | LangGraph + MCP + APIM Gateway |

### Citadel Integration

All agent API traffic routes through **Layer 1 (Governance Hub)**. Enterprise knowledge retrieval is governed by **Layer 4 (Security Fabric)** to ensure permission-respecting access. **Layer 3 (Agent Identity)** registers the agent with a unique identity and lifecycle policy.

## Scenario 2: Data and Analytics Agents

### Challenge

Dashboards exist everywhere, but employees still ask analysts to pull numbers. People know data exists but struggle to find the right report or interpret what they see. Analysts spend time answering ad-hoc questions instead of doing strategic analysis.

### Opportunity

Let employees ask questions in natural language and get answers grounded in actual business data. *"What were Q3 sales by region?"* returns an answer from your data warehouse. Analysts reclaim time for high-value work.

### Builder Approaches

| Persona | Approach | Toolchain |
|---------|----------|-----------|
| **Business User** | Use native BI copilot integrations to ask natural language questions over existing datasets. No building required. | BI platform natural language interface |
| **Data Engineer** | Build data agents that expose curated datasets for natural language queries. Define the semantic layer — business terms, relationships, and metrics. | Custom data agent + Business Data layer |
| **Developer** | Create analytics agents that expose APIs for embedding in AI applications and customer-facing solutions. | LangChain + vector DB + APIM Gateway |

### Citadel Integration

**Layer 2 (AI Control Plane)** captures data access telemetry and monitors agent reasoning over business data. Compliance dashboards track which datasets agents consume across the fleet.

## Scenario 3: Sales Lead Engagement

### Challenge

Sellers spend hours researching prospects before outreach, or skip research and send generic messages that get ignored. The information exists in CRM, news feeds, and relationship data — but assembling it for each prospect is manual and slow.

### Opportunity

Deploy agents that research prospects, identify warm paths through existing relationships, and draft personalized outreach grounded in real context. Sellers engage with confidence. Response rates climb.

### Builder Approaches

| Persona | Approach | Toolchain |
|---------|----------|-----------|
| **Power User** | Build prospecting agents that pull relationship signals from workspace systems, match against ICPs in business data, and enrich with company research via the enterprise knowledge layer. | Low-code builder + CRM connectors |
| **Developer** | Build agents that integrate with marketing automation platforms, apply custom lead scoring models, and orchestrate multi-step research workflows. | LangGraph + MCP tools + APIM Gateway |

### Citadel Integration

All MCP server calls to CRM and marketing systems are gated through **Layer 1 (Governance Hub)** for rate limiting and content policy. **Layer 4 (Security Fabric)** governs PII access in prospect data.

---

## Scenario 4: Customer Support and Product Feedback

### Challenge

The real value of support isn't just resolving tickets — it's closing the loop. Feature requests and bug reports come through support channels, get logged somewhere, and then stall. Engineering teams don't have visibility into what customers are actually asking for.

### Opportunity

Deploy agents that handle frontline support grounded in product knowledge and resolution history. When a customer reports an issue, the agent triages, enriches with context, and creates a tracked issue automatically. On the engineering side, coding agents draft implementation plans — accelerating the path from customer voice to product improvement.

### Builder Approaches

| Persona | Approach | Toolchain |
|---------|----------|-----------|
| **Power User** | Build support agents grounded in product docs via RAG. Configure triage workflows that enrich issues with customer context and create tracked issues automatically. | Low-code builder + Enterprise Knowledge layer |
| **Developer** | Build agents that monitor issue trackers, draft implementation plans, propose code changes, and route to human reviewers. Integrate coding agents into CI/CD workflows. | LangGraph + Claude Agent SDK + code tools |

### Citadel Integration

**Layer 3 (Agent Identity)** ensures coding agents have scoped access to only the repositories and systems they need. **Layer 2 (AI Control Plane)** evaluates agent outputs for quality and safety before they reach human reviewers.

## Scenario Summary

| Scenario | Primary Personas | Key Intelligence Layers | Citadel Focus |
|----------|------------------|------------------------|---------------|

| **Data and Analytics** | Business User, Data Engineer, Developer | Business Data | Layer 2 observability |
| **Sales Lead Engagement** | Power User, Developer | Workspace + Business Data + Knowledge | Layer 1 MCP gateway, Layer 4 PII |
