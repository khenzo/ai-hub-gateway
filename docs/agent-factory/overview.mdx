# Enterprise Agent Factory

The **Enterprise Agent Factory** is an open operating model for deploying AI agents at scale. It brings together intelligence layers, builder toolchains, and governance capabilities into a unified approach that scales from experimentation to production — independent of any single vendor or framework.

## The Agentic Shift

The next decade of enterprise software will be shaped by adaptive AI agents — systems that can reason, learn, and act across business processes. Adoption is accelerating: agents promise to extend an organization's capacity beyond human-only cost, speed, and scale. They interpret information, apply judgment, and produce high-quality outputs that previously required significant human effort.

In reality, scaling agents from experimentation to production requires more than tooling. It requires:

- A **disciplined operating model** with clear governance
- **Intelligence layers** that ground agents in enterprise context
- A **harness architecture** that controls how agents act safely
- A **governance foundation** that enforces policy at runtime

The Enterprise Agent Factory provides all four.

## Three Pillars

| Pillar | Description |
|--------|-------------|
| **Intelligence Layers** | Workspace, business data, and enterprise knowledge layers provide the contextual grounding agents need to operate effectively |
| **Builder Toolchains** | Low-code platforms, open frameworks (LangChain, LangGraph, Claude Agent SDK, OpenAI Agents SDK, Microsoft Agent Framework), and SDK-based approaches match the right tool to every builder persona |
| **Harness & Governance** | The Citadel 4-layer architecture provides identity, security, compliance, and operational telemetry at scale |

## The Three-Layer Agent Architecture

Modern production agents are runtime environments for reasoning systems — not just prompts wrapped around a model. The Enterprise Agent Factory is built on a three-layer architecture:

```
┌─────────────────────────────────────────────────────────────┐
│               ENTERPRISE AGENT FACTORY                       │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  REASONING LAYER  (LLM + Orchestration Engine)       │    │
│  │  Plans, reasons, selects tools, generates outputs    │    │
│  └───────────────────────┬─────────────────────────────┘    │
│                          │                                   │
│  ┌───────────────────────▼─────────────────────────────┐    │
│  │  CONTEXT LAYER  (Memory, Skills, Intelligence)       │    │
│  │  Filesystem memory · Progressive skills · RAG        │    │
│  └───────────────────────┬─────────────────────────────┘    │
│                          │                                   │
│  ┌───────────────────────▼─────────────────────────────┐    │
│  │  EXECUTION LAYER  (Tools, Middleware, Sandbox)        │    │
│  │  Tool interfaces · HITL hooks · Isolation sandbox     │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

| Layer | Responsibility | Key Mechanisms |
|-------|---------------|----------------|
| **Reasoning** | Plans, decides, and generates | LLM inference, orchestration graphs |
| **Context** | Controls what the model sees | Working memory, agent skills, RAG |
| **Execution** | Controls how the model acts | Tools, middleware, HITL, sandboxes |

## Intelligence Layers

Agents are only as useful as the context they can access. The Agent Factory builds on three interconnected intelligence layers:

```
┌─────────────────────────────────────────────────────────────┐
│                    INTELLIGENCE LAYERS                        │
│                                                              │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────┐  │
│  │  Workspace      │  │  Business Data  │  │ Enterprise  │  │
│  │  Intelligence   │  │  Intelligence   │  │ Knowledge   │  │
│  │  (People/Work)  │  │  (Operations)   │  │ (All Data)  │  │
│  └─────────────────┘  └─────────────────┘  └─────────────┘  │
│         │                     │                   │          │
│         └─────────────────────┴───────────────────┘          │
│                               ▼                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │         Agent Harness + Citadel Governance           │    │
│  │  Tools · Skills · Middleware · APIM Gateway          │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

| Layer | Focus | Sources |
|-------|-------|---------|
| **Workspace Intelligence** | How people work — communication, tasks, collaboration | Calendars, messaging, documents, project systems |
| **Business Data Intelligence** | How the business operates — metrics, KPIs, processes | Data warehouses, BI platforms, operational databases |
| **Enterprise Knowledge Intelligence** | All institutional knowledge — structured and unstructured | Internal docs, SOPs, MCP-connected systems, public web |

## Context Engineering & Harness Engineering

The two foundational disciplines of production agent development:

**Context Engineering** controls *what the model sees* — managing the LLM context window as a dynamic memory hierarchy rather than a static prompt. Mechanisms include filesystem working memory, progressive skill disclosure, and automatic summarization.

**Harness Engineering** controls *how the model acts* — building infrastructure that constrains what agents can physically do. The model never executes anything directly; every action passes through a tool interface that the harness can validate, log, approve, or sandbox.

Together these disciplines separate an experimental demo from a production system.

## Citadel Governance Foundation

The Enterprise Agent Factory deploys on the **Citadel 4-Layer Architecture** — an open governance foundation built on Azure API Management that enforces policy, identity, and security regardless of which agent frameworks or models are used:

| Citadel Layer | Agent Factory Role |
|---------------|-------------------|
| **Layer 1: Governance Hub** | Routes all agent API calls through APIM; enforces rate limiting, auth, and content policy |
| **Layer 2: AI Control Plane** | Captures execution traces, evaluations, and compliance telemetry for agent workloads |
| **Layer 3: Agent Identity** | Registers agent identities, manages lifecycle, and enforces access packages |
| **Layer 4: Security Fabric** | Threat detection, data governance, PII protection across all intelligence sources |
