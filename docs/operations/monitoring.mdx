# Monitoring and Observability

This page covers the operations-focused view of monitoring Citadel infrastructure and AI workloads. For agent-specific observability — evaluations, CI/CD quality gates, red teaming, and production runbooks — see the dedicated guide below.

<Note>
  For the complete observability guide covering the five best practices, evaluator types, CI/CD integration, red teaming, and production runbooks, see [Agent Observability](/agent-factory/agent-observability).
</Note>

## What the Agent Observability Guide Covers

The [Agent Observability](/agent-factory/agent-observability) page provides a comprehensive runbook-style guide for monitoring AI agents in production on Azure AI Foundry and Citadel. Topics include:

- **Model selection** — using Azure AI Foundry leaderboards to pick the right model before deployment
- **Continuous evaluation** — agent, quality, and risk/safety evaluators running in development and production
- **CI/CD quality gates** — automated evaluation checks via GitHub Actions and Azure DevOps extensions
- **AI red teaming** — adversarial testing with the Azure AI Red Teaming Agent and PyRIT before every promotion
- **Production monitoring dashboard** — Azure Monitor Application Insights, Azure Workbooks, and governance integrations
