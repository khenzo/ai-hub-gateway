# Citadel Sizing

This guide provides recommendations for sizing the AI Citadel Governance Hub based on various deployment scenarios. Proper sizing ensures optimal performance, scalability, and cost-efficiency.

The approach is based on T-Shirt Sizing Methodology, which categorizes deployments into Small, Medium, and Large sizes based on expected workloads and user counts.

<Note>
Below estimates are based on typical usage patterns and may vary based on specific customer requirements and workloads. It is advisable to conduct a detailed assessment for precise sizing.
</Note>

## Governance Hub Environment Sizing

As a centralized supplemental landing zone for AI workloads, the AI Citadel Governance Hub can be sized based on the number of environments required by the customer. Common configurations include:

- One central for production and non-production
- Two with one for production and one for non-production
- Other: based on customer requirements (like have 3 or more environments)

## T-Shirt Sizing Categories

### Small: Development and Experimental

No SLA – development only. Can be multiple environments per customer. Full network isolation supported.

| Component | Configuration |
|-----------|---------------|
| **APIM** | Developer SKU, 1 unit |
| **Cosmos DB** | 400 RU/s |
| **Event Hub** | 1 capacity unit |
| **Estimated Capacity** | Experimental, low load |
| **Cost Estimate** | [Azure Calculator](https://azure.com/e/b23669e7f59948c488608ecab4b7c1fa) |

**Best For:**
- Proof of concepts
- Development environments
- Testing and validation
- Cost-conscious experimentation

### Medium Classic: Non-Production with SLA

Minimum-production or non-production with SLA. Can be multiple environments per customer. Full network isolation supported (uses APIM Premium).

| Component | Configuration |
|-----------|---------------|
| **APIM** | Premium SKU, 1 unit |
| **Cosmos DB** | 1000 RU/s |
| **Event Hub** | 2 capacity units |
| **Estimated Capacity** | ~300-500 PTU |
| **Cost Estimate** | [Azure Calculator](https://azure.com/e/7a539b8ef73a4b5e87b5e60c079f0835) |

**Best For:**
- Staging environments
- Production with moderate load
- High availability requirements

### Medium V2: Non-Production with SLA

Minimum-production or non-production with SLA. Can be multiple environments per customer. Full network isolation supported (uses APIM Standard V2).

| Component | Configuration |
|-----------|---------------|
| **APIM** | StandardV2 SKU, 1-2 units |
| **Cosmos DB** | 1000 RU/s |
| **Event Hub** | 2 capacity units |
| **Estimated Capacity** | ~200 PTU per APIM unit |
| **Cost Estimate** | [Azure Calculator](https://azure.com/e/b44716c27c054c4eb979248744bc341d) |

**Best For:**
- Modern workloads preferring V2 SKUs
- Cost-optimized production
- Scalable architectures

### Large: Multi-Zone Production

Multi-zone production setup with SLA. Full network isolation supported.

| Component | Configuration |
|-----------|---------------|
| **APIM** | PremiumV2 SKU, 2+ units |
| **Cosmos DB** | Autoscale 4000+ RU/s |
| **Event Hub** | 4+ capacity units |
| **Estimated Capacity** | ~1,000-1,500 PTU |
| **Cost Estimate** | [Azure Calculator](https://azure.com/e/fed48a9999804396a7e60d31550f24bf) |

**Best For:**
- Enterprise production
- High availability across zones
- Large-scale deployments
- Mission-critical workloads

## Considerations for Sizing

When determining the appropriate size for the AI Citadel Governance Hub, consider the following factors:

<AccordionGroup>
  <Accordion title="Traffic Patterns">
- Peak requests per second (RPS)
- Average token count per request
- Concurrent user sessions
- Burst traffic patterns
  </Accordion>
  <Accordion title="Model Requirements">
- Number of models to support
- Model complexity (GPT-4o vs GPT-5)
- Mix of PTU vs PAYG models
- Geographic distribution needs
  </Accordion>
  <Accordion title="Governance Needs">
- PII processing requirements
- Content safety volume
- Logging and audit retention
- Number of access contracts
  </Accordion>
  <Accordion title="Growth Projections">
- 6-month growth forecast
- New team onboarding plans
- Model expansion roadmap
- Multi-region requirements
  </Accordion>
</AccordionGroup>

<Note>
Above calculation excludes the sizing for the agentic applications (which would be sized separately based on their own requirements) and LLM and other AI services which should be sized based on the expected usage patterns and workloads.
</Note>

Medium and Medium V2 tiers can be scaled out by adding additional APIM units to handle increased load as per customer requirements.

## SKU Implementation

Based on the selected T-Shirt and environment, update the SKUs in the deployment <a href="/bicep/infra/main.bicepparam" target="_blank">main.bicepparam</a> under the `COMPUTE SKU & SIZE` section:

```bicep
// ============================================================================
// COMPUTE SKU & SIZE - SKUs and capacity settings for services
// ============================================================================
param apimSku = readEnvironmentVariable('APIM_SKU', 'StandardV2')
param apimSkuUnits = int(readEnvironmentVariable('APIM_SKU_UNITS', '1'))
param eventHubCapacityUnits = int(readEnvironmentVariable('EVENTHUB_CAPACITY', '1'))
param cosmosDbRUs = int(readEnvironmentVariable('COSMOS_DB_RUS', '400'))
param logicAppsSkuCapacityUnits = int(readEnvironmentVariable('LOGIC_APPS_SKU_CAPACITY_UNITS', '1'))
param languageServiceSkuName = readEnvironmentVariable('LANGUAGE_SERVICE_SKU_NAME', 'S')
param aiContentSafetySkuName = readEnvironmentVariable('AI_CONTENT_SAFETY_SKU_NAME', 'S0')
param apicSku = readEnvironmentVariable('APIC_SKU', 'Free')
param keyVaultSkuName = readEnvironmentVariable('KEY_VAULT_SKU_NAME', 'standard')
```

<Note>
Above is the default parameters configuration. It needs to be adjusted based on the selections made based on final sizing.
</Note>

## Scaling Guidelines

### Vertical Scaling (Upgrade SKU)

| From | To | When |
|------|-----|------|
| Developer | StandardV2 | Moving from dev to production |
| StandardV2 | PremiumV2 | Need multi-zone HA |
| Premium (Classic) | PremiumV2 | Modernizing deployment |

### Horizontal Scaling (Add Units)

Add APIM units when:
- Consistent CPU > 70%
- Response times increasing
- Throttling events frequent
- Planned traffic growth

<Warning>
Developer SKU cannot be scaled horizontally. Must upgrade to StandardV2 or Premium first.
</Warning>

## Cost Optimization Tips

1. **Start with smaller SKUs** - Scale up based on actual usage
2. **Use dev/test pricing** - Azure dev/test subscriptions offer discounts
3. **Reserved capacity** - Pre-purchase Cosmos DB RUs for savings
4. **Right-size Event Hub** - Match capacity to actual throughput
5. **Monitor and adjust** - Review metrics monthly and right-size
