# Bicep Modules Reference

This reference documents all Bicep modules available in the AI Landing Zone (Citadel Governance Hub). Each module is designed for reuse, composability, and environment-specific customization.

<Note>
All modules are located in the `bicep/infra/` directory of the [ai-hub-gateway repository](https://github.com/Azure/ai-hub-gateway-solution-accelerator). Refer to the [Deployment Guide](/guides/deployment-guide) for deployment sequence.
</Note>

---

## Module Architecture

### Design Principles

The Citadel Governance Hub uses a layered module architecture:

1. **Root template** (`main.bicep`) — Orchestrates all modules via feature flags
2. **Domain modules** — Grouped by functional area (networking, APIM, AI)
3. **Shared modules** — Reused components (Key Vault secrets, private endpoints, managed identity)
4. **Specialty deployments** — Standalone deployments for specific use cases

### Module Dependency Graph

```
main.bicep
├── modules/networking/vnet.bicep          (Hub VNet)
│   ├── modules/networking/subnet.bicep
│   └── modules/networking/dns.bicep
├── modules/apim/apim.bicep               (API Management)
│   ├── modules/apim/api.bicep
│   ├── modules/apim/llm-backends.bicep
│   └── modules/apim/policy-fragments.bicep
├── modules/monitor/monitoring.bicep       (Observability)
│   ├── modules/monitor/loganalytics.bicep
│   └── modules/monitor/applicationinsights.bicep
├── modules/keyvault/keyvault.bicep        (Key Vault)
└── modules/ai/cognitiveservices.bicep     (Azure OpenAI)
```

Specialty deployments (standalone):
- `citadel-access-contracts/` — Access contract lifecycle management
- `llm-backend-onboarding/` — LLM backend registration
- `foundry-integration/` — AI Foundry connectivity
- `apim-gateway-upgrade/` — APIM upgrade without redeployment

### Versioning Strategy

Modules follow semantic versioning embedded in parameter comments:
- **Major changes**: New required parameters or breaking changes
- **Minor changes**: New optional parameters with defaults
- **Patch changes**: Bug fixes, no parameter changes

The root `main.bicep` pins specific module versions. When upgrading, review the changelog in each module's header comments.

### Customization Guidelines

To customize a module without forking:

1. Use optional parameters with defaults for deviations
2. Override via `.bicepparam` files — never modify core modules
3. Create wrapper modules that call core modules with custom parameters
4. Contribute enhancements back via pull requests

## Core Modules Reference

### Network Module (`modules/networking/vnet.bicep`)

**Purpose:** Deploys the Hub VNet with subnets, NSGs, and optional DDoS protection. This is the network foundation for all Citadel components.

**Required Parameters:**

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `location` | string | Azure region | `eastus` |
| `vnetName` | string | Name of the Virtual Network | `vnet-citadel-hub-prod` |
| `addressPrefixes` | array | VNet address spaces | `['10.0.0.0/16']` |
| `subnets` | array | Subnet configurations | See below |

**Optional Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `enableDdosProtection` | bool | `false` | Enable DDoS Protection Plan |
| `ddosPlanId` | string | `''` | Resource ID of existing DDoS plan |
| `dnsServers` | array | `[]` | Custom DNS servers (empty = Azure DNS) |
| `tags` | object | `{}` | Resource tags |

**Subnet Configuration Object:**

```bicep
// Subnet configuration array element
{
  name: 'snet-apim'
  addressPrefix: '10.0.1.0/24'
  serviceEndpoints: ['Microsoft.KeyVault']
  privateEndpointNetworkPolicies: 'Disabled'
  privateLinkServiceNetworkPolicies: 'Enabled'
  delegations: []
  nsgRules: [
    {
      name: 'AllowHTTPS'
      priority: 100
      direction: 'Inbound'
      access: 'Allow'
      protocol: 'Tcp'
      sourcePortRange: '*'
      destinationPortRange: '443'
      sourceAddressPrefix: '*'
      destinationAddressPrefix: '*'
    }
  ]
}
```

**Outputs:**

| Output | Type | Description |
|--------|------|-------------|
| `vnetId` | string | Resource ID of the VNet |
| `vnetName` | string | Name of the VNet |
| `subnetIds` | object | Map of subnet name to resource ID |

**Usage Example:**

```bicep
module hubVnet 'modules/networking/vnet.bicep' = {
  name: 'hubVnetDeploy'
  params: {
    location: location
    vnetName: 'vnet-citadel-hub-${environmentName}'
    addressPrefixes: ['10.0.0.0/16']
    subnets: [
      {
        name: 'snet-apim'
        addressPrefix: '10.0.1.0/24'
        privateEndpointNetworkPolicies: 'Disabled'
      }
      {
        name: 'snet-private-endpoints'
        addressPrefix: '10.0.3.0/24'
        privateEndpointNetworkPolicies: 'Disabled'
      }
    ]
    tags: commonTags
  }
}

// Reference subnet IDs in downstream modules
var apimSubnetId = hubVnet.outputs.subnetIds['snet-apim']
```

### API Management Module (`modules/apim/apim.bicep`)

**Purpose:** Deploys Azure API Management with full VNet integration, managed identity, diagnostic settings, and optional custom domain configuration. This is the core AI gateway component.

**Required Parameters:**

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `location` | string | Azure region | `eastus` |
| `apimName` | string | APIM service instance name | `apim-citadel-prod` |
| `apimSku` | string | Service tier | `Premium` |
| `apimSkuCount` | int | Scale units | `2` |
| `publisherEmail` | string | API publisher contact | `admin@contoso.com` |
| `publisherName` | string | Organization name | `Contoso` |
| `subnetId` | string | Resource ID of APIM subnet | `/subscriptions/.../subnets/snet-apim` |
| `logAnalyticsWorkspaceId` | string | Log Analytics workspace ID | `/subscriptions/...` |

**Optional Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `virtualNetworkType` | string | `'Internal'` | `Internal` or `External` VNet mode |
| `customDomains` | array | `[]` | Custom hostname configurations |
| `certificates` | array | `[]` | SSL certificates from Key Vault |
| `additionalLocations` | array | `[]` | Additional regions (Premium multi-region) |
| `enableClientCertificate` | bool | `false` | Require client certificates |
| `minApiVersion` | string | `'2021-08-01'` | Minimum REST API version |
| `systemAssignedIdentity` | bool | `true` | Enable system-assigned managed identity |
| `userAssignedIdentityId` | string | `''` | User-assigned managed identity resource ID |

**Allowed SKU Values:** `Developer`, `Basic`, `Standard`, `StandardV2`, `Premium`

**SKU Comparison:**

| SKU | VNet Integration | Multi-region | SLA | Use Case |
|-----|-----------------|-------------|-----|---------|
| `Developer` | Internal only | No | None | Dev/test |
| `Standard` | External only | No | 99.9% | Small workloads |
| `StandardV2` | Both | No | 99.95% | Mid-size workloads |
| `Premium` | Both | Yes | 99.99% | Production |

**Outputs:**

| Output | Type | Description |
|--------|------|-------------|
| `apimId` | string | Resource ID of APIM |
| `apimName` | string | Name of APIM service |
| `gatewayUrl` | string | API gateway URL |
| `managementApiUrl` | string | Management API URL |
| `principalId` | string | System-assigned managed identity principal ID |

**Usage Example:**

```bicep
module apim 'modules/apim/apim.bicep' = {
  name: 'apimDeploy'
  params: {
    location: location
    apimName: 'apim-citadel-${environmentName}'
    apimSku: isProd ? 'Premium' : 'Developer'
    apimSkuCount: isProd ? 2 : 1
    publisherEmail: publisherEmail
    publisherName: publisherName
    subnetId: hubVnet.outputs.subnetIds['snet-apim']
    logAnalyticsWorkspaceId: monitoring.outputs.logAnalyticsWorkspaceId
    virtualNetworkType: 'Internal'
    additionalLocations: isProd ? [
      {
        location: 'westus2'
        sku: { name: 'Premium', capacity: 1 }
      }
    ] : []
    tags: commonTags
  }
}
```

### LLM Backend Modules (`modules/apim/llm-backends.bicep`)

**Purpose:** Registers AI service backends (Azure OpenAI, AI Foundry) in APIM for intelligent routing, load balancing, and failover.

**Required Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `apimName` | string | APIM service name |
| `resourceGroupName` | string | APIM resource group |
| `backends` | array | List of LLM backend configurations |

**Backend Configuration Object:**

```bicep
// Single backend configuration
{
  name: 'aoai-eastus-gpt4'
  description: 'Azure OpenAI East US - GPT-4'
  url: 'https://<openai-name>.openai.azure.com/openai'
  protocol: 'http'
  resourceId: '/subscriptions/.../cognitiveservices/openai-eastus'
  circuitBreakerRules: {
    name: 'openai-breaker'
    failureCondition: {
      count: 3
      interval: 'PT10S'
      percentage: 50
      statusCodeRanges: [{ min: 429, max: 429 }, { min: 500, max: 599 }]
    }
    tripDuration: 'PT30S'
  }
}
```

**Usage Example:**

```bicep
module llmBackends 'modules/apim/llm-backends.bicep' = {
  name: 'llmBackendsDeploy'
  params: {
    apimName: apim.outputs.apimName
    resourceGroupName: resourceGroup().name
    backends: [
      {
        name: 'aoai-primary'
        url: 'https://${openAIPrimary.name}.openai.azure.com/openai'
        resourceId: openAIPrimary.id
      }
      {
        name: 'aoai-secondary'
        url: 'https://${openAISecondary.name}.openai.azure.com/openai'
        resourceId: openAISecondary.id
      }
    ]
  }
}
```

### Governance Module — Access Contracts (`citadel-access-contracts/main.bicep`)

**Purpose:** Deploys AI Access Contracts — the governance constructs that define per-workload AI usage policies, rate limits, content safety, and observability. Each contract deploys as an APIM Product with associated policies.

**Required Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `apimName` | string | Target APIM service name |
| `apimResourceGroup` | string | APIM resource group |
| `contracts` | array | List of access contract definitions |

**Optional Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `defaultContentSafetyEnabled` | bool | `true` | Apply Content Safety to all contracts |
| `defaultPIIProtectionEnabled` | bool | `false` | Apply PII detection to all contracts |
| `kvName` | string | `''` | Key Vault for storing subscription keys |
| `logAnalyticsWorkspaceId` | string | `''` | Workspace for audit logging |

**Access Contract Definition Object:**

```bicep
// Access contract configuration
{
  name: 'hr-chat-agent'
  displayName: 'HR Chat Agent'
  description: 'Policy for HR department AI chat assistant'
  state: 'published'

  // Token quotas and rate limits
  tokenQuota: {
    calls: 1000
    renewalPeriod: 3600    // Per hour
  }
  rateLimit: {
    calls: 100
    renewalPeriod: 60      // Per minute
  }

  // Allowed AI models
  allowedModels: ['gpt-4', 'gpt-4o-mini']

  // Security policies
  contentSafetyCategories: {
    hate: 'medium'
    selfHarm: 'high'
    sexual: 'high'
    violence: 'medium'
  }
  piiProtectionEnabled: true
  piiAction: 'anonymize'   // 'anonymize' | 'block'

  // Subscriber configuration
  subscriptionRequired: true
  approvalRequired: false   // Self-service subscription
}
```

**Outputs:**

| Output | Type | Description |
|--------|------|-------------|
| `productIds` | array | APIM product IDs created |
| `subscriptionKeys` | object | Map of contract name to subscription key (stored in KV) |

**Usage Example:**

```bicep
module accessContracts 'citadel-access-contracts/main.bicep' = {
  name: 'accessContractsDeploy'
  params: {
    apimName: apim.outputs.apimName
    apimResourceGroup: resourceGroup().name
    defaultContentSafetyEnabled: true
    kvName: keyVault.outputs.keyVaultName
    logAnalyticsWorkspaceId: monitoring.outputs.logAnalyticsWorkspaceId
    contracts: [
      {
        name: 'hr-chat-agent'
        displayName: 'HR Chat Agent'
        allowedModels: ['gpt-4o-mini']
        rateLimit: { calls: 50, renewalPeriod: 60 }
      }
      {
        name: 'sales-assistant'
        displayName: 'Sales Assistant'
        allowedModels: ['gpt-4', 'gpt-4o-mini']
        rateLimit: { calls: 100, renewalPeriod: 60 }
        piiProtectionEnabled: true
      }
    ]
  }
}
```

### Foundry Integration Module (`foundry-integration/main.bicep`)

**Purpose:** Configures APIM to route traffic to Azure AI Foundry deployments, supporting dynamic model discovery, static model registration, and custom authentication.

**Required Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `apimName` | string | APIM service name |
| `apimResourceGroup` | string | APIM resource group |
| `foundryEndpoint` | string | AI Foundry project endpoint URL |
| `integrationMode` | string | `'static'` or `'dynamic'` |

**Optional Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `staticModels` | array | `[]` | Model list for static mode |
| `customAuthEnabled` | bool | `false` | Use custom auth headers |
| `customHeaders` | object | `{}` | Custom HTTP headers to forward |
| `managedIdentityId` | string | `''` | Managed identity for Foundry authentication |

**Integration Modes:**

- **Static mode**: Explicit list of available models — predictable, faster routing
- **Dynamic mode**: Discovers models from Foundry at runtime — flexible, adds latency

**Usage Example:**

```bicep
module foundryIntegration 'foundry-integration/main.bicep' = {
  name: 'foundryIntegrationDeploy'
  params: {
    apimName: apim.outputs.apimName
    apimResourceGroup: resourceGroup().name
    foundryEndpoint: 'https://<foundry-project>.api.azureml.ms'
    integrationMode: 'static'
    staticModels: [
      { name: 'gpt-4o', deploymentName: 'gpt4o-prod' }
      { name: 'text-embedding-ada-002', deploymentName: 'embed-prod' }
    ]
    managedIdentityId: apim.outputs.principalId
  }
}
```

### Production Environment Override (`main.parameters.prod.bicepparam`)

using './main.bicep'

### Parameter Validation Rules

Bicep enforces these validation rules:

```bicep
@allowed(['Developer', 'Basic', 'Standard', 'StandardV2', 'Premium'])

@minValue(1)
@maxValue(12)

@minLength(3)
@maxLength(50)
@description('Environment name used in resource naming')

@minLength(1)
@description('Azure region for primary deployment')
param location string
```

## Composition Patterns

### Standard Hub Deployment

Minimal viable hub with APIM and monitoring:

```bicep
// Standard hub deployment composition
module network 'modules/networking/vnet.bicep' = { ... }
module apim 'modules/apim/apim.bicep' = {
  dependsOn: [network]
  ...
}
module monitoring 'modules/monitor/monitoring.bicep' = { ... }
module keyVault 'modules/keyvault/keyvault.bicep' = { ... }
module privateEndpoints 'modules/networking/private-endpoint.bicep' = {
  dependsOn: [apim, keyVault]
  ...
}
```

### Multi-Region Production Deployment

Deploy APIM Premium across two regions for 99.99% SLA:

```bicep
// Production multi-region pattern
module apim 'modules/apim/apim.bicep' = {
  params: {
    apimSku: 'Premium'
    apimSkuCount: 2
    additionalLocations: [
      {
        location: secondaryLocation
        sku: { name: 'Premium', capacity: 1 }
        virtualNetworkConfiguration: {
          subnetResourceId: secondarySubnet.id
        }
      }
    ]
  }
}
```

### Spoke-per-Team Pattern

Deploy isolated spoke for each business unit or team:

```bicep
// Spoke factory pattern
var teamSpokes = [
  { name: 'hr', addressPrefix: '10.1.0.0/16' }
  { name: 'finance', addressPrefix: '10.2.0.0/16' }
  { name: 'sales', addressPrefix: '10.3.0.0/16' }
]

module spokes 'modules/networking/vnet.bicep' = [for spoke in teamSpokes: {
  name: 'spoke-${spoke.name}-deploy'
  params: {
    vnetName: 'vnet-spoke-${spoke.name}-${environmentName}'
    addressPrefixes: [spoke.addressPrefix]
  }
}]
```

### DR/HA Configuration

Configure active-passive DR with automated failover:

```bicep
// HA configuration using APIM backend pools
module llmBackends 'modules/apim/llm-backend-pools.bicep' = {
  params: {
    apimName: apim.outputs.apimName
    pools: [
      {
        name: 'gpt4-pool'
        backends: [
          { name: 'aoai-eastus', weight: 100, isBackup: false }
          { name: 'aoai-westus2', weight: 0, isBackup: true }
        ]
      }
    ]
  }
}
```

## Customization Guide

### Extending Modules

Add custom configuration to existing modules without forking:

```bicep
// Wrapper module pattern
// file: custom-modules/apim-with-custom-domains.bicep

module baseApim '../modules/apim/apim.bicep' = {
  name: 'baseApimDeploy'
  params: {
    // Pass through all required params
    ...apimBaseParams
    // Add custom domain configuration
    customDomains: [
      {
        type: 'Proxy'
        hostName: 'api.contoso.com'
        keyVaultId: customCertKeyVaultUri
      }
    ]
  }
}
```

### Creating Custom Access Contract Policies

Create organization-specific APIM policy fragments:

```xml
<!-- custom-policies/org-compliance-policy.xml -->
<fragment>
  <!-- Organization-specific compliance header -->
  <set-header name="X-Org-Compliance-ID" exists-action="override">
    <value>@(context.Subscription.Id + "-" + context.RequestId)</value>
  </set-header>

  <!-- Custom audit logging -->
  <log-to-eventhub logger-id="compliance-logger">
    @{
      return new JObject(
        new JProperty("requestId", context.RequestId),
        new JProperty("userId", context.User?.Id),
        new JProperty("operation", context.Operation.Name),
        new JProperty("timestamp", DateTime.UtcNow)
      ).ToString();
    }
  </log-to-eventhub>
</fragment>
```

Register the custom policy fragment via Bicep:

```bicep
module customPolicyFragment 'modules/apim/policy-fragments.bicep' = {
  params: {
    apimName: apim.outputs.apimName
    fragmentName: 'org-compliance'
    fragmentDescription: 'Organization compliance audit logging'
    fragmentValue: loadTextContent('custom-policies/org-compliance-policy.xml')
  }
}
```

### Testing Custom Modules

Validate custom Bicep modules before deployment:

```bash
# 1. Validate Bicep syntax
az bicep build --file custom-modules/my-module.bicep

# 2. Run what-if deployment (no changes made)
az deployment sub what-if \
  --location eastus \
  --template-file main.bicep \
  --parameters my-custom.bicepparam

# 3. Validate parameter file
az bicep build-params --file my-custom.bicepparam

# 4. Run integration test in dev subscription
az deployment sub create \
  --name "test-$(date +%Y%m%d%H%M%S)" \
  --location eastus \
  --template-file main.bicep \
  --parameters my-custom.bicepparam \
  --confirm-with-what-if
```

### Module Contribution Guidelines

To contribute a new module or improvement:

1. Follow the existing parameter naming conventions (`camelCase`)
2. Include `@description()` decorators on all parameters
3. Include `@allowed()` constraints on enum parameters
4. Always output resource IDs and names (minimum outputs)
5. Include module-level `// PARAMETERS` and `// RESOURCES` comments
6. Test in both dev (Developer SKU) and prod-like (Premium SKU) configurations
7. Update `anatomy.md` with new module description
