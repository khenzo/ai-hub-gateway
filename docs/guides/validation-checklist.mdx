# Validation Checklist

This checklist verifies that the AI Landing Zone (Citadel Governance Hub) is correctly deployed, functioning, and ready for production use. Complete all sections before declaring the deployment successful.

<Note>
Complete the [Deployment Checklist](/guides/deployment-checklist) before starting validation. If any validation step fails, consult the troubleshooting guidance linked at the bottom of each section.
</Note>

---

## How to Use This Checklist

Run each validation command from an appropriate location:
- **Azure CLI commands**: Run from any machine with Azure CLI and appropriate role
- **Network connectivity tests**: Run from a VM inside the Hub VNet (jumpbox or bastion)
- **API tests**: Run from within the VNet or via APIM portal test console

**Status symbols:**
- `[ ]` — Not validated
- `[P]` — Passed
- `[F]` — Failed (document failure and resolution in notes)
- `[S]` — Skipped (document reason)

**Validation record:**

| Environment | Validator | Date | Overall Status |
|-------------|-----------|------|----------------|
| Development | | | |
| Staging | | | |
| Production | | | |

## Section 1: Network Validation

### 1.1 Hub VNet Connectivity

| # | Validation | Command / Method | Expected Result | Status | Notes |
|---|-----------|-----------------|-----------------|--------|-------|
| 1.1.1 | Hub VNet exists and is in Succeeded state | `az network vnet show --name vnet-hub-<env> --resource-group rg-citadel-hub-<env> --query provisioningState` | `Succeeded` | `[ ]` | |
| 1.1.2 | APIM subnet has correct address prefix | `az network vnet subnet show --vnet-name vnet-hub-<env> -g rg-citadel-hub-<env> --name snet-apim --query addressPrefix` | Matches planned CIDR | `[ ]` | |
| 1.1.3 | Private endpoint subnet configured | `az network vnet subnet show --name snet-private-endpoints ...` | Subnet exists | `[ ]` | |
| 1.1.4 | NSGs associated with subnets | `az network vnet subnet list --vnet-name vnet-hub-<env> -g rg-citadel-hub-<env> --query "[].{name:name, nsg:networkSecurityGroup.id}"` | NSG IDs populated | `[ ]` | |

### 1.2 Hub-Spoke Connectivity

| # | Validation | Command / Method | Expected Result | Status | Notes |
|---|-----------|-----------------|-----------------|--------|-------|
| 1.2.1 | Hub-to-spoke peering in Connected state | `az network vnet peering list --resource-group rg-citadel-hub-<env> --vnet-name vnet-hub-<env> --query "[].{name:name, state:peeringState}"` | `Connected` for all spokes | `[ ]` | |
| 1.2.2 | Spoke-to-hub peering in Connected state | `az network vnet peering list --resource-group rg-citadel-spoke<n>-<env> --vnet-name vnet-spoke<n>-<env> --query "[].peeringState"` | `Connected` | `[ ]` | |
| 1.2.3 | Bidirectional ping: hub to spoke | From hub VM: `ping 10.1.1.4` (spoke VM IP) | Successful ping | `[ ]` | Requires jumpbox |
| 1.2.4 | Bidirectional ping: spoke to hub | From spoke VM: `ping 10.0.1.4` (hub APIM IP) | Successful ping | `[ ]` | |
| 1.2.5 | Cross-spoke isolation confirmed | From spoke1 VM: attempt `ping <spoke2-ip>` | No response (isolated) | `[ ]` | Spokes should not route to each other |

## Validation Failure Reference

If validation steps fail, use this reference to diagnose:

| Section | Common Failure | Likely Cause | Quick Fix |
|---------|---------------|--------------|-----------|
| 1.2 | VNet peering not Connected | Peering only created one direction | Create peering in both directions |
| 1.3 | DNS resolves to public IP | Private DNS zone not linked to VNet | Link DNS zone to hub VNet |
| 2.1 | APIM gateway unreachable | NSG blocking port 3443 | Add NSG rule for `ApiManagement` service tag |
| 2.3 | Auth returns 500 (not 401) | APIM policy XML error | Check APIM policy XML in developer portal |
| 3.1 | OpenAI accessible from public internet | Public network access not disabled | Set `publicNetworkAccess: Disabled` on OpenAI resource |
| 4.1 | No logs in Log Analytics | Diagnostic settings not configured | Enable diagnostics on APIM and OpenAI |
| 5.1 | High latency | Cold network connection or wrong region | Test from within VNet; verify same region |

For unresolved failures, consult the [Troubleshooting Guide](/guides/troubleshooting) or open a GitHub issue on the ai-hub-gateway repository.
