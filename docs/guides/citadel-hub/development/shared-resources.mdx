# Shared Resources

The `shared/` directory contains Python utilities and code snippets used across validation notebooks and automation scripts.

## Overview

These utilities provide common functionality for:
- Azure CLI command execution
- API Management interactions
- HTTP request patterns
- OpenAI API testing
- Output formatting and logging

## Core Utilities (`utils.py`)

### Print Helpers

Color-coded output functions for consistent logging:

```python
from shared.utils import print_info, print_ok, print_error, print_warning

print_info("Starting deployment...")
print_ok("Deployment completed", output="Resource group created")
print_warning("High latency detected")
print_error("Connection failed", output="Timeout after 30s")
```

| Function | Usage | Output |
|----------|-------|--------|
| `print_info(msg)` | Information messages | 👉🏽 Blue text |
| `print_ok(msg, output, duration)` | Success messages | ✅ Green text |
| `print_warning(msg)` | Warning messages | ⚠️ Yellow text |
| `print_error(msg, output, duration)` | Error messages | ❌ Red text |
| `print_command(cmd)` | Command execution | ⚙️ Blue text |

### Command Execution

Execute Azure CLI commands with structured output:

```python
from shared.utils import run

# Execute Azure CLI command
output = run(
    "az group list --output json",
    "Retrieved resource groups",
    "Failed to list resource groups"
)

if output.success:
    print(f"Found {len(output.json_data)} resource groups")
```

**Output Class Properties:**
- `success` (bool): Command execution status
- `text` (str): Raw command output
- `json_data` (dict/list): Parsed JSON output

### Sensitive Data Masking

Automatically mask sensitive values in data structures:

```python
from shared.utils import mask_sensitive_values

config = {
    "apiKey": "sk-1234567890",
    "endpoint": "https://api.example.com",
    "secret": "my-secret-value"
}

masked = mask_sensitive_values(config)
# Result: {"apiKey": "########", "endpoint": "https://api.example.com", "secret": "########"}
```

## APIM Tools (`apimtools.py`)

### APIM Client Tool

High-level client for Azure API Management operations:

```python
from shared.apimtools import APIMClientTool

# Initialize client
apim = APIMClientTool(
    resource_group_name="rg-citadel-prod",
    apim_resource_name="apim-citadel"
)

# Initialize connection
apim.initialize()

# Get gateway URL
print(f"Gateway URL: {apim.apim_resource_gateway_url}")
```

### Discovering APIs

Find APIs matching specific path filters:

```python
# Discover OpenAI API
api_path = apim.discover_api(api_path_filter="/openai")

if api_path:
    print(f"Found API at: {api_path}")
```

### Managing Subscriptions

Access subscription keys programmatically:

```python
# Get all subscriptions with keys
for sub in apim.apim_subscriptions:
    print(f"Subscription: {sub['name']}")
    print(f"Key: {sub['key'][:10]}...")
```

## Code Snippets

The `shared/snippets/` directory contains reusable code blocks for Jupyter notebooks.

### Using Snippets

Load snippets in notebooks using IPython's `%load` magic:

```python
# %load ../../shared/snippets/openai-api-requests.py
import time
from openai import AzureOpenAI

# ... loaded code ...
```

<Note>
After loading, the `%load` command becomes a comment with the loaded code below it. This provides static snapshots that are maintained even if the snippet changes.
</Note>

### Available Snippets

| Snippet | Purpose |
|---------|---------|
| `api-http-requests.py` | Generic HTTP request patterns |
| `openai-api-requests.py` | Azure OpenAI API interactions |
| `create-az-deployment.py` | Azure deployment creation helpers |
| `deployment-outputs.py` | Deployment output parsing |
| `create-az-resource-group.py` | Resource group management |

### Maintaining Snippets

When editing notebooks that use snippets:

1. Remove all code below the `%load` line
2. Uncomment the `%load` command
3. Execute the cell to reload fresh code
4. Test the changes
5. Check git for diffs
6. Commit if working correctly
