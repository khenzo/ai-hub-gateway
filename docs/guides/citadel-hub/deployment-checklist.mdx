# Deployment Checklist

This checklist guides deployment teams through every step required for a complete AI Landing Zone (Citadel Governance Hub) deployment. Work through each section in order and sign off each item before proceeding to the next phase.

<Note>
Print or copy this checklist to a tracking document (Confluence, Azure DevOps, etc.) and assign owners to each item before starting deployment. See the [Deployment Guide](/guides/deployment-guide) for detailed instructions on each step.
</Note>

---

## How to Use This Checklist

**Before starting:** Assign an owner to each checklist section and agree on a RACI matrix (see [Roles and Responsibilities](/getting-started/roles-and-responsibilities)).

**Tracking symbols:**
- `[ ]` — Not started
- `[~]` — In progress
- `[x]` — Complete
- `[!]` — Blocked (document blocker in notes column)

**Deployment environments:** Run this checklist once per environment (dev, staging, prod).

| Environment | Owner | Date Started | Date Completed |

## Common Deployment Issues

| Issue | Cause | Resolution |
|-------|-------|-----------|
| APIM deployment timeout | APIM Premium takes 30-45 min | Wait and verify: `az apim show --query provisioningState` |
| NSG blocking APIM management | Port 3443 not open to `ApiManagement` service tag | Add NSG rule: Allow 3443 from `ApiManagement` tag |
| Private DNS not resolving | DNS zone not linked to VNet | Link private DNS zone to hub VNet |
| Service principal lacks permissions | Missing User Access Administrator | Grant role at subscription scope |
| Quota exceeded for APIM | Single Premium unit default limit | Request limit increase via Support ticket |
