# 🛡️ Content Safety Integration

## Overview

**Azure AI Content Safety** provides AI-powered content moderation capabilities that detect and filter harmful content across text and images. When integrated with Azure API Management in an AI Landing Zone, it becomes a critical Layer 4 (Security Fabric) component that enforces responsible AI policies at the gateway level.

## What is Azure AI Content Safety

Azure AI Content Safety is a content moderation service that uses machine learning models to analyze content for potential harm across multiple categories:

| Category | Description | Use Cases |
|----------|-------------|-----------|
| **Hate** | Content expressing hatred or discrimination | Community platforms, social media |
| **Sexual** | Sexually explicit or suggestive content | Customer service, public APIs |
| **Violence** | Content depicting or encouraging violence | Gaming, content platforms |
| **Self-Harm** | Content encouraging self-injury | Mental health, wellness apps |

### Severity Levels and Thresholds

Content Safety returns severity scores from 0 to 3:

| Level | Score | Action | Example |
|-------|-------|--------|---------|
| **Safe** | 0 | Allow | Normal conversation |
| **Low** | 1 | Log/Flag | Borderline content |
| **Medium** | 2 | Review/Block | Potentially harmful |
| **High** | 3 | Block | Clearly harmful |

### Request/Response Interception Patterns

Content Safety can be applied at different stages:

| Stage | Pattern | Use Case |
|-------|---------|----------|
| **Pre-processing** | Check user input before AI processing | Prevent harmful prompts |
| **Post-processing** | Check AI response before delivery | Filter AI-generated content |
| **Both** | Check both request and response | Maximum safety coverage |

### Async vs Sync Moderation

| Mode | Latency | Use Case | Implementation |
|------|---------|----------|----------------|
| **Synchronous** | 50-200ms | Real-time APIs, chat | APIM policy blocking |
| **Asynchronous** | Background | Batch processing, logging | Queue-based analysis |

## Implementation Patterns

### APIM Policy Integration

#### Basic Content Safety Check

```xml
<policies>
  <inbound>
    <!-- Extract prompt from OpenAI request -->
    <set-variable name="prompt" value="@{
      JObject body = context.Request.Body.As<JObject>(preserveContent: true);
      return body?["messages"]?.Last?["content"]?.ToString() ?? "";
    }" />
    
    <!-- Send to Content Safety API -->
    <send-request mode="new" response-variable-name="safety-check" timeout="5">
      <set-url>https://{{content-safety-name}}.cognitiveservices.azure.com/contentsafety/text:analyze?api-version=2023-10-01</set-url>
      <set-method>POST</set-method>
      <set-header name="Ocp-Apim-Subscription-Key">
        <value>{{content-safety-key}}</value>
      </set-header>
      <set-header name="Content-Type">
        <value>application/json</value>
      </set-header>
      <set-body>@{
        return new JObject(
          new JProperty("text", context.Variables.GetValueOrDefault("prompt", "")),
          new JProperty("categories", new JArray("Hate", "Sexual", "Violence", "SelfHarm")),
          new JProperty("blocklistNames", new JArray()),
          new JProperty("haltOnBlocklistHit", true)
        ).ToString();
      }</set-body>
    </send-request>
    
    <!-- Check safety results -->
    <set-variable name="safety-result" value="@(((IResponse)context.Variables["safety-check"]).Body.As<JObject>())" />
    
    <choose>
      <when condition="@{
        var result = context.Variables.GetValueOrDefault<JObject>("safety-result", null);
        if (result == null) return false;
        var categories = result["categoriesAnalysis"] as JArray;
        if (categories == null) return false;
        foreach (var cat in categories) {
          var severity = cat["severity"]?.Value<int>() ?? 0;
          if (severity >= 2) return true; // Medium or High
        }
        return false;
      }">
        <return-response>
          <set-status code="400" reason="Content Safety Violation" />
          <set-header name="Content-Type">
            <value>application/json</value>
          </set-header>
          <set-body>{
            "error": "Content violates safety policy",
            "violation": "@(context.Variables.GetValueOrDefault<JObject>("safety-result", null)?["categoriesAnalysis"]?.ToString())"
          }</set-body>
        </return-response>
      </when>
    </choose>
  </inbound>
</policies>
```

#### Advanced Content Safety with Custom Categories

```xml
<policies>
  <inbound>
    <!-- Check content with custom thresholds -->
    <send-request mode="new" response-variable-name="safety-check" timeout="5">
      <set-url>https://{{content-safety-name}}.cognitiveservices.azure.com/contentsafety/text:analyze?api-version=2023-10-01</set-url>
      <set-method>POST</set-method>
      <set-header name="Ocp-Apim-Subscription-Key">
        <value>{{content-safety-key}}</value>
      </set-header>
      <set-header name="Content-Type">
        <value>application/json</value>
      </set-header>
      <set-body>@{
        return new JObject(
          new JProperty("text", context.Variables.GetValueOrDefault("prompt", "")),
          new JProperty("categories", new JArray("Hate", "Sexual", "Violence", "SelfHarm")),
          new JProperty("blocklistNames", new JArray("profanity-list", "pii-patterns")),
          new JProperty("haltOnBlocklistHit", true),
          new JProperty("outputType", "FourSeverityLevels")
        ).ToString();
      }</set-body>
    </send-request>
    
    <set-variable name="safety-result" value="@(((IResponse)context.Variables["safety-check"]).Body.As<JObject>())" />
    
    <!-- Log all safety checks for audit -->
    <log-to-eventhub logger-id="content-safety-logger">
      @{
        return new JObject(
          new JProperty("timestamp", DateTime.UtcNow.ToString("o")),
          new JProperty("type", "content-safety-check"),
          new JProperty("contractId", context.Variables.GetValueOrDefault("contract-id", "")),
          new JProperty("result", context.Variables.GetValueOrDefault<JObject>("safety-result", null)?.ToString()),
          new JProperty("action", "pre-processing")
        ).ToString();
      }
    </log-to-eventhub>
    
    <!-- Block on Medium or High severity -->
    <choose>
      <when condition="@{
        var result = context.Variables.GetValueOrDefault<JObject>("safety-result", null);
        if (result == null) return true; // Fail safe
        
        var categories = result["categoriesAnalysis"] as JArray;
        if (categories == null) return true;
        
        foreach (var cat in categories) {
          var category = cat["category"]?.ToString();
          var severity = cat["severity"]?.Value<int>() ?? 0;
          
          // Different thresholds per category
          if (category == "SelfHarm" && severity >= 1) return true; // Strict for self-harm
          if (category == "Violence" && severity >= 2) return true; // Medium+ for violence
          if (category == "Hate" && severity >= 2) return true;
          if (category == "Sexual" && severity >= 2) return true;
        }
        return false;
      }">
        <return-response>
          <set-status code="400" reason="Content Safety Violation" />
          <set-header name="Content-Type">
            <value>application/json</value>
          </set-header>
          <set-body>{
            "error": "Content violates safety policy",
            "code": "CONTENT_SAFETY_VIOLATION",
            "message": "The submitted content has been flagged by our safety system. Please review and modify your request."
          }</set-body>
        </return-response>
      </when>
    </choose>
  </inbound>
</policies>
```

### Post-Processing Response Filtering

```xml
<policies>
  <outbound>
    <!-- Check AI response for safety -->
    <set-variable name="response-content" value="@{
      JObject body = context.Response.Body.As<JObject>(preserveContent: true);
      return body?["choices"]?.First?["message"]?.["content"]?.ToString() ?? "";
    }" />
    
    <!-- Send to Content Safety -->
    <send-request mode="new" response-variable-name="response-safety" timeout="5">
      <set-url>https://{{content-safety-name}}.cognitiveservices.azure.com/contentsafety/text:analyze?api-version=2023-10-01</set-url>
      <set-method>POST</set-method>
      <set-header name="Ocp-Apim-Subscription-Key">
        <value>{{content-safety-key}}</value>
      </set-header>
      <set-header name="Content-Type">
        <value>application/json</value>
      </set-header>
      <set-body>@{
        return new JObject(
          new JProperty("text", context.Variables.GetValueOrDefault("response-content", "")),
          new JProperty("categories", new JArray("Hate", "Sexual", "Violence", "SelfHarm"))
        ).ToString();
      }</set-body>
    </send-request>
    
    <!-- Mask harmful content in response -->
    <choose>
      <when condition="@{
        var result = ((IResponse)context.Variables["response-safety"]).Body.As<JObject>();
        var categories = result["categoriesAnalysis"] as JArray;
        if (categories == null) return false;
        foreach (var cat in categories) {
          if (cat["severity"]?.Value<int>() ?? 0 >= 2) return true;
        }
        return false;
      }">
        <set-body>@{
          JObject body = context.Response.Body.As<JObject>(preserveContent: true);
          body["choices"][0]["message"]["content"] = "[Content filtered by safety policy]";
          return body.ToString();
        }</set-body>
      </when>
    </choose>
  </outbound>
</policies>
```

### API Integration Code Sample

```csharp
using Azure;
using Azure.AI.ContentSafety;

public class ContentSafetyService
{
    private readonly ContentSafetyClient _client;
    
    public ContentSafetyService(string endpoint, string key)
    {
        _client = new ContentSafetyClient(
            new Uri(endpoint), 
            new AzureKeyCredential(key));
    }
    
    public async Task<SafetyResult> AnalyzeTextAsync(string text)
    {
        var request = new AnalyzeTextOptions(text);
        
        // Analyze all categories
        request.Categories.Add(TextCategory.Hate);
        request.Categories.Add(TextCategory.Sexual);
        request.Categories.Add(TextCategory.Violence);
        request.Categories.Add(TextCategory.SelfHarm);
        
        var response = await _client.AnalyzeTextAsync(request);
        
        return new SafetyResult
        {
            IsSafe = response.Value.CategoriesAnalysis.All(
                c => c.Severity < 2),
            Categories = response.Value.CategoriesAnalysis.Select(
                c => new CategoryResult
                {
                    Category = c.Category.ToString(),
                    Severity = c.Severity ?? 0
                }).ToList()
        };
    }
}
```

## Policy Enforcement

### Block vs Flag Decisions

| Action | Threshold | Use Case | Implementation |
|--------|-----------|----------|----------------|
| **Block** | Severity ≥ 2 | Production APIs, strict compliance | Return 400 error |
| **Flag** | Severity ≥ 1 | Logging, review queues | Log and allow |
| **Allow** | Severity = 0 | All safe content | Pass through |

### Batching Strategies

```csharp
// Batch multiple content checks
public async Task<BatchResult> AnalyzeBatchAsync(List<string> texts)
{
    var tasks = texts.Select(async text =>
    {
        var result = await AnalyzeTextAsync(text);
        return new { Text = text, Result = result };
    });
    
    var results = await Task.WhenAll(tasks);
    return new BatchResult { Results = results.ToList() };
}
```

## Best Practices

### Threshold Tuning

1. **Start Conservative**: Begin with severity ≥ 2 blocking
2. **Monitor False Positives**: Track blocked requests and user complaints
3. **Industry-Specific**: Adjust thresholds based on use case
4. **Gradual Relaxation**: Slowly lower thresholds as confidence increases

### False Positive Handling

| Strategy | Implementation |
|----------|----------------|
| **Appeal Process** | Allow users to request manual review |
| **Allow Lists** | Maintain lists of approved terms/phrases |
| **Context Awareness** | Consider conversation context in decisions |
| **Feedback Loop** | Report false positives to improve models |
