# 🔒 PII Protection

## Overview

**Personally Identifiable Information (PII)** represents any data that could identify an individual. In AI workloads, PII protection is critical for regulatory compliance, data security, and maintaining user trust. This document covers comprehensive patterns for detecting, classifying, and protecting PII in AI Landing Zones.

## PII Protection Overview

### Types of PII in AI Workloads

| Category | Examples | Risk Level |
|----------|----------|------------|
| **Personal Identifiers** | Name, SSN, Passport number | Critical |
| **Contact Information** | Email, phone, address | High |
| **Financial Data** | Credit card, bank account | Critical |
| **Health Information** | Medical records, diagnoses | Critical |
| **Biometric Data** | Fingerprints, facial recognition | Critical |
| **Online Identifiers** | IP address, cookies, device IDs | Medium |
| **Professional Data** | Employee ID, work email | Medium |

## PII Handling Strategies

### Redaction and Masking

| Strategy | Example Input | Example Output | Use Case |
|----------|---------------|----------------|----------|
| **Full Redaction** | "John's SSN is 123-45-6789" | "John's SSN is [REDACTED]" | High-sensitivity PII |
| **Partial Masking** | "john.doe@email.com" | "j***.***@email.com" | Email addresses |
| **Token Replacement** | "123-45-6789" | `<SSN\_001>` | Analytics, training |
| **Hashing** | "john.doe@email.com" | "a1b2c3d4..." | Pseudonymization |

```xml
<!-- APIM policy for PII redaction -->
<policies>
  <outbound>
    <set-body>@{
      JObject response = context.Response.Body.As<JObject>(preserveContent: true);
      string content = response.ToString();
      
      // Redact SSNs
      content = Regex.Replace(content, @"\b\d{3}-\d{2}-\d{4}\b", "[SSN-REDACTED]");
      
      // Mask email addresses
      content = Regex.Replace(content, 
        @"\b([A-Za-z0-9._%+-])[^@]*@([A-Za-z0-9.-]+\.[A-Z|a-z]{2,})\b", 
        "$1***@$2");
      
      // Mask phone numbers
      content = Regex.Replace(content, 
        @"\b\d{3}-\d{3}-\d{4}\b", 
        "***-***-****");
      
      // Redact credit cards
      content = Regex.Replace(content, 
        @"\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b", 
        "[CREDIT-CARD-REDACTED]");
      
      return content;
    }</set-body>
  </outbound>
</policies>
```

### Tokenization Approaches

```
Tokenization Architecture:

┌─────────────────────────────────────────────────────────────┐
│                     TOKEN VAULT                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   Original Data              Token                           │
│   ─────────────              ─────                           │
│   john.doe@email.com    →    tok_abc123xyz                   │
│   123-45-6789           →    tok_ssn_789def                  │
│   555-123-4567          →    tok_phn_456ghi                  │
│                                                              │
│   Token Format: tok_<type>_<random>                          │
│                                                              │
│   Properties:                                                │
│   • Non-reversible (no algorithm to derive original)         │
│   • Format-preserving (same length, valid format)            │
│   • Vault-secured (Key Vault encryption)                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

```csharp
// Tokenization service
public class TokenizationService
{
    private readonly ITokenVault _vault;
    
    public async Task<string> TokenizeAsync(
        string originalValue, 
        PiiType type)
    {
        // Generate unique token
        var token = $"tok_{type.ToString().ToLower()}_{Guid.NewGuid().ToString("N")[..12]}";
        
        // Store mapping in vault
        await _vault.StoreAsync(new TokenMapping
        {
            Token = token,
            OriginalValue = await EncryptAsync(originalValue),
            Type = type,
            CreatedAt = DateTime.UtcNow
        });
        
        return token;
    }
    
    public async Task<string> DetokenizeAsync(string token)
    {
        var mapping = await _vault.RetrieveAsync(token);
        if (mapping == null) return null;
        
        return await DecryptAsync(mapping.OriginalValue);
    }
}
```

## Implementation Patterns

### Pre-Processing Detection

```xml
<policies>
  <inbound>
    <!-- Extract request content -->
    <set-variable name="request-content" value="@{
      JObject body = context.Request.Body.As<JObject>(preserveContent: true);
      return body?["messages"]?.Last?["content"]?.ToString() ?? "";
    }" />
    
    <!-- Send to Text Analytics for PII detection -->
    <send-request mode="new" response-variable-name="pii-detection" timeout="10">
      <set-url>https://{{text-analytics-endpoint}}.cognitiveservices.azure.com/text/analytics/v3.1/entities/recognition/pii</set-url>
      <set-method>POST</set-method>
      <set-header name="Ocp-Apim-Subscription-Key">
        <value>{{text-analytics-key}}</value>
      </set-header>
      <set-header name="Content-Type">
        <value>application/json</value>
      </set-header>
      <set-body>@{
        return new JObject(
          new JProperty("documents", new JArray(
            new JObject(
              new JProperty("id", "1"),
              new JProperty("text", context.Variables.GetValueOrDefault("request-content", "")),
              new JProperty("language", "en")
            )
          ))
        ).ToString();
      }</set-body>
    </send-request>
    
    <!-- Check for high-confidence PII -->
    <set-variable name="pii-result" value="@(((IResponse)context.Variables["pii-detection"]).Body.As<JObject>())" />
    
    <choose>
      <when condition="@{
        var result = context.Variables.GetValueOrDefault<JObject>("pii-result", null);
        if (result == null) return false;
        var documents = result["documents"] as JArray;
        if (documents == null || documents.Count == 0) return false;
        var entities = documents[0]["entities"] as JArray;
        if (entities == null) return false;
        // Check for high-confidence PII
        return entities.Any(e => 
          (e["confidenceScore"]?.Value<double>() ?? 0) > 0.8 &&
          new[] { "SocialSecurityNumber", "CreditCardNumber", "PassportNumber" }
            .Contains(e["category"]?.ToString()));
      }">
        <return-response>
          <set-status code="400" reason="PII Detected" />
          <set-header name="Content-Type">
            <value>application/json</value>
          </set-header>
          <set-body>{
            "error": "Request contains sensitive PII",
            "message": "Please remove personal identifiers before submitting.",
            "code": "PII_DETECTED"
          }</set-body>
        </return-response>
      </when>
    </choose>
  </inbound>
</policies>
```

### Post-Processing Redaction

```xml
<policies>
  <outbound>
    <!-- Detect PII in AI response -->
    <set-variable name="response-text" value="@{
      JObject body = context.Response.Body.As<JObject>(preserveContent: true);
      return body?["choices"]?.First?["message"]?.["content"]?.ToString() ?? "";
    }" />
    
    <!-- Send to PII detection service -->
    <send-request mode="new" response-variable-name="response-pii" timeout="10">
      <set-url>https://{{text-analytics-endpoint}}.cognitiveservices.azure.com/text/analytics/v3.1/entities/recognition/pii</set-url>
      <set-method>POST</set-method>
      <set-header name="Ocp-Apim-Subscription-Key">
        <value>{{text-analytics-key}}</value>
      </set-header>
      <set-body>@{
        return new JObject(
          new JProperty("documents", new JArray(
            new JObject(
              new JProperty("id", "1"),
              new JProperty("text", context.Variables.GetValueOrDefault("response-text", "")),
              new JProperty("language", "en")
            )
          ))
        ).ToString();
      }</set-body>
    </send-request>
    
    <!-- Redact detected PII from response -->
    <set-body>@{
      JObject body = context.Response.Body.As<JObject>(preserveContent: true);
      string content = body?["choices"]?.First?["message"]?.["content"]?.ToString() ?? "";
      
      var piiResult = ((IResponse)context.Variables["response-pii"]).Body.As<JObject>();
      var documents = piiResult["documents"] as JArray;
      
      if (documents != null && documents.Count > 0)
      {
        var entities = documents[0]["entities"] as JArray;
        if (entities != null)
        {
          // Sort by offset descending to avoid index shifting
          var sortedEntities = entities
            .OrderByDescending(e => e["offset"]?.Value<int>() ?? 0)
            .ToList();
          
          foreach (var entity in sortedEntities)
          {
            int offset = entity["offset"]?.Value<int>() ?? 0;
            int length = entity["length"]?.Value<int>() ?? 0;
            string category = entity["category"]?.ToString() ?? "PII";
            
            if (offset >= 0 && length > 0 && offset + length <= content.Length)
            {
              content = content.Substring(0, offset) + 
                       $"[{category}-REDACTED]" + 
                       content.Substring(offset + length);
            }
          }
        }
      }
      
      body["choices"][0]["message"]["content"] = content;
      return body.ToString();
    }</set-body>
  </outbound>
</policies>
```

### Data Subject Request Handling

```csharp
// Data Subject Access Request (DSAR) handler
public class DataSubjectRequestHandler
{
    private readonly IVectorStore _vectorStore;
    private readonly IAuditLogRepository _auditRepo;
    
    public async Task<DsarResult> ProcessAccessRequestAsync(
        string dataSubjectId)
    {
        // Search all AI systems for subject's data
        var vectorResults = await _vectorStore.SearchAsync(
            filter: $"userId eq '{dataSubjectId}'");
        
        var auditLogs = await _auditRepo.QueryAsync(
            x => x.UserId == dataSubjectId);
        
        return new DsarResult
        {
            SubjectId = dataSubjectId,
            VectorStoreRecords = vectorResults,
            AuditLogEntries = auditLogs,
            GeneratedAt = DateTime.UtcNow
        };
    }
    
    public async Task<bool> ProcessDeletionRequestAsync(
        string dataSubjectId)
    {
        // Delete from vector store
        await _vectorStore.DeleteByUserIdAsync(dataSubjectId);
        
        // Anonymize audit logs (retain for compliance)
        await _auditRepo.AnonymizeAsync(dataSubjectId);
        
        // Delete any cached conversations
        await _cache.DeleteAsync($"conv:{dataSubjectId}");
        
        return true;
    }
}
```

## Compliance Alignment

### GDPR Data Protection

| GDPR Article | Requirement | Implementation |
|--------------|-------------|----------------|
| **Art. 5** | Principles (lawfulness, purpose limitation) | Data minimization, purpose tagging |
| **Art. 6** | Lawful basis | Consent management, legitimate interest |
| **Art. 17** | Right to erasure | Data deletion workflows |
| **Art. 20** | Data portability | Export in machine-readable format |
| **Art. 25** | Privacy by design | Default PII protection |
| **Art. 32** | Security of processing | Encryption, access controls |
| **Art. 35** | DPIA | Risk assessment for high-risk processing |

```
GDPR Compliance Checklist:

□ Data minimization implemented (collect only necessary PII)
□ Consent tracking for AI training data
□ Right to deletion workflows operational
□ Data portability API available
□ Privacy impact assessments documented
□ DPO contact information published
□ Breach notification procedures in place
□ Cross-border transfer mechanisms (SCCs)
```

### PCI DSS Considerations

For AI workloads handling payment data:

```
PCI DSS Scope Reduction:

Before: AI system in PCI scope
┌──────────┐    ┌──────────┐    ┌──────────┐
│  Client  │───▶│   AI     │───▶│ Payment  │
│          │    │  System  │    │ Gateway  │
└──────────┘    └──────────┘    └──────────┘
                     ↑
                All in scope

After: Tokenization reduces scope
┌──────────┐    ┌──────────┐    ┌──────────┐
│  Client  │───▶│Tokenizer │───▶│ Payment  │
│          │    │ (In Scope)│   │ Gateway  │
└──────────┘    └────┬─────┘    └──────────┘
                     │
                     ▼
               ┌──────────┐
               │    AI    │
               │  System  │
               │(Out of   │
               │ Scope)   │
               └──────────┘
```
