# 🔐 Security and Compliance

## Overview

Security for AI workloads requires a **defense in depth** approach that addresses unique threats while maintaining compliance with regulatory frameworks. This document establishes the security foundation for AI Landing Zones, integrating Layer 4 Security Fabric capabilities with enterprise governance requirements.

### Zero Trust for AI Workloads

| Principle | Traditional | Zero Trust |
|-----------|-------------|------------|
| **Assume Breach** | Trust internal network | Verify every request |
| **Least Privilege** | Broad access | Minimum necessary |
| **Verify Explicitly** | Implicit trust | Continuous validation |
| **Micro-segmentation** | Flat network | Isolated segments |

## Threat Protection

### Prompt Injection Detection

```
Prompt Injection Attack Types:

1. DIRECT INJECTION
   "Ignore previous instructions and..."
   
2. INDIRECT INJECTION
   Hidden in user content, documents
   
3. JAILBREAK ATTEMPTS
   "DAN" (Do Anything Now) patterns
   
4. SYSTEM PROMPT LEAKAGE
   Attempts to reveal instructions
```

#### Detection Patterns

```xml
<!-- APIM policy for prompt injection detection -->
<policies>
  <inbound>
    <set-variable name="prompt" value="@{
      JObject body = context.Request.Body.As<JObject>(preserveContent: true);
      return body?["messages"]?.Last?["content"]?.ToString() ?? "";
    }" />
    
    <!-- Check for injection patterns -->
    <choose>
      <when condition="@{
        var prompt = context.Variables.GetValueOrDefault("prompt", "").ToLower();
        var patterns = new[] {
          "ignore previous",
          "ignore instructions",
          "system prompt",
          "dan mode",
          "jailbreak",
          "do anything now"
        };
        return patterns.Any(p => prompt.Contains(p));
      }">
        <return-response>
          <set-status code="400" reason="Potential Injection Detected" />
          <set-header name="Content-Type">
            <value>application/json</value>
          </set-header>
          <set-body>{
            "error": "Request contains potentially malicious patterns",
            "code": "PROMPT_INJECTION_DETECTED"
          }</set-body>
        </return-response>
      </when>
    </choose>
  </inbound>
</policies>
```
