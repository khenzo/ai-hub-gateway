## Citadel Gateway

The **Citadel Gateway** is built on Azure API Management and serves as the unified entry point for all AI service requests. It provides a consistent interface for applications while enforcing security and governance policies.

### Key Capabilities

<AccordionGroup>
  <Accordion title="Authentication & Authorization">
    - Azure AD integration for token validation
    - OAuth 2.0 and managed identity support
    - API key management for legacy applications
    - Role-based access control (RBAC)
  </Accordion>
  
  <Accordion title="Rate Limiting & Throttling">
    - Per-application request quotas
    - Token consumption limits
    - Burst handling and queue management
    - Graceful degradation under load
  </Accordion>
  
  <Accordion title="Request Transformation">
    - Header injection for tracing
    - Payload transformation
    - Backend routing and load balancing
    - Protocol translation
  </Accordion>
  
  <Accordion title="Caching">
    - Response caching for repeated queries
    - Token usage optimization
    - Backend load reduction
  </Accordion>
</AccordionGroup>

### Architecture Pattern

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Application   │────▶│  Citadel Gateway │────▶│  AI Service     │
│                 │     │  (APIM + Policy) │     │  (OpenAI, etc.) │
└─────────────────┘     └──────────────────┘     └─────────────────┘
                                │
                                ▼
                       ┌──────────────────┐
                       │  Policy Engine   │
                       │  (Validation)    │
                       └──────────────────┘
```

## AI Service Registry

The **AI Service Registry** maintains a catalog of all AI services available through Citadel, along with their access contracts and configurations.

### Registry Contents

<Steps>
  <Step title="Service Definitions">
    - Available models (GPT-4, GPT-3.5, etc.)
    - Service endpoints and versions
    - Capability descriptions
    - Usage guidelines
  </Step>
  
  <Step title="Access Contracts">
    - Rate limits per application
    - Cost budgets and allocations
    - Content filter configurations
    - Data residency requirements
  </Step>
  
  <Step title="Application Mappings">
    - Which apps can access which services
    - Contract assignments
    - Approval workflows
  </Step>
</Steps>

### Access Contract Example

```json
{
  "contractId": "app-prod-001",
  "application": "Customer Support Bot",
  "services": ["gpt-4", "text-embedding-ada-002"],
  "limits": {
    "requestsPerMinute": 100,
    "tokensPerDay": 1000000,
    "monthlyBudget": 500
  },
  "filters": {
    "contentSafety": "strict",
    "allowedTopics": ["support", "general"]
  }
}
```

  <Card title="Logs" icon="file-lines">

## Policy Engine

The **Policy Engine** enforces governance policies and access contracts across all AI service consumption.

### Policy Types

<Steps>
  <Step title="Security Policies">
    - Authentication requirements
    - Encryption in transit
    - Private endpoint enforcement
    - Data classification rules
  </Step>
  
  <Step title="Operational Policies">
    - Rate limiting and quotas
    - Circuit breaker patterns
    - Retry and timeout settings
    - Resource tagging requirements
  </Step>
  
  <Step title="Compliance Policies">
    - Content safety filters
    - Data residency restrictions
    - Audit logging requirements
    - Retention policies
  </Step>
  
  <Step title="Cost Policies">
    - Budget thresholds and alerts
    - Cost allocation rules
    - Spending limits per application
    - Reserved capacity management
  </Step>
</Steps>

### Policy Enforcement Points

Policies are enforced at multiple points in the request flow:

1. **Gateway Entry**: Authentication, rate limiting
2. **Registry Lookup**: Access validation
3. **Request Processing**: Content filtering
4. **Response Handling**: Audit logging
