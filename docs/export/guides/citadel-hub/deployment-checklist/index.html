    Deployment Checklist

    Pre-deployment and deployment phase checklists for the AI Landing Zone (Citadel Governance Hub)

    ​
    Deployment Checklist

    This checklist guides deployment teams through every step required for a complete AI Landing Zone (Citadel Governance Hub) deployment. Work through each section in order and sign off each item before proceeding to the next phase.
    Print or copy this checklist to a tracking document (Confluence, Azure DevOps, etc.) and assign owners to each item before starting deployment. See the Deployment Guide for detailed instructions on each step.

    ​
    How to Use This Checklist

    Before starting: Assign an owner to each checklist section and agree on a RACI matrix (see Roles and Responsibilities). Tracking symbols:
    • [ ] — Not started
    • [~] — In progress
    • [x] — Complete
    • [!] — Blocked (document blocker in notes column)
    Deployment environments: Run this checklist once per environment (dev, staging, prod).
    EnvironmentOwnerDate StartedDate Completed
    Development
    Staging
    Production

    Common Deployment Issues

    IssueCauseResolution
    APIM deployment timeoutAPIM Premium takes 30-45 minWait and verify: az apim show --query provisioningState
    NSG blocking APIM managementPort 3443 not open to ApiManagement service tagAdd NSG rule: Allow 3443 from ApiManagement tag
    Private DNS not resolvingDNS zone not linked to VNetLink private DNS zone to hub VNet
    Service principal lacks permissionsMissing User Access AdministratorGrant role at subscription scope
    Quota exceeded for APIMSingle Premium unit default limitRequest limit increase via Support ticket
