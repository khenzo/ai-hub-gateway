    Validation Checklist

    Post-deployment validation procedures to confirm successful AI Landing Zone deployment

    ​
    Validation Checklist

    This checklist verifies that the AI Landing Zone (Citadel Governance Hub) is correctly deployed, functioning, and ready for production use. Complete all sections before declaring the deployment successful.
    Complete the Deployment Checklist before starting validation. If any validation step fails, consult the troubleshooting guidance linked at the bottom of each section.

    ​
    How to Use This Checklist

    Run each validation command from an appropriate location:
    • Azure CLI commands: Run from any machine with Azure CLI and appropriate role
    • Network connectivity tests: Run from a VM inside the Hub VNet (jumpbox or bastion)
    • API tests: Run from within the VNet or via APIM portal test console
    Status symbols:
    • [ ] — Not validated
    • [P] — Passed
    • [F] — Failed (document failure and resolution in notes)
    • [S] — Skipped (document reason)
    Validation record:
    EnvironmentValidatorDateOverall Status

    Validation Failure Reference

    If validation steps fail, use this reference to diagnose:
    SectionCommon FailureLikely CauseQuick Fix
    1.2VNet peering not ConnectedPeering only created one directionCreate peering in both directions
    1.3DNS resolves to public IPPrivate DNS zone not linked to VNetLink DNS zone to hub VNet
    2.1APIM gateway unreachableNSG blocking port 3443Add NSG rule for ApiManagement service tag
    2.3Auth returns 500 (not 401)APIM policy XML errorCheck APIM policy XML in developer portal
    3.1OpenAI accessible from public internetPublic network access not disabledSet publicNetworkAccess: Disabled on OpenAI resource
    4.1No logs in Log AnalyticsDiagnostic settings not configuredEnable diagnostics on APIM and OpenAI
    5.1High latencyCold network connection or wrong regionTest from within VNet; verify same region
    For unresolved failures, consult the Troubleshooting Guide or open a GitHub issue on the ai-hub-gateway repository.
