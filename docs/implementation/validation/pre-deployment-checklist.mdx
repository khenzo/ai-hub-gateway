# Pre-Deployment Checklist

Complete this checklist before deploying AI Landing Zone infrastructure to catch configuration errors, permission issues, and resource constraints early in the deployment process.

<Note>
This checklist should be completed after architecture design and before running any deployment commands. Allow 2-4 hours for thorough validation.
</Note>

---

#### Required Azure RBAC Roles

Verify the deploying identity has these roles:

```bash
# Check your current roles at subscription scope
DEPLOYER_ID=$(az ad signed-in-user show --query "id" -o tsv)
sub_id=$(az account show --query "id" -o tsv)

echo "Current role assignments:"
az role assignment list \
  --assignee "$DEPLOYER_ID" \
  --scope "/subscriptions/$sub_id" \
  --query "[].{role:roleDefinitionName, scope:scope}" \
  --output table
```

**Required minimum roles:**

| Role | Purpose | Scope |
|------|---------|-------|
| **Contributor** | Deploy and manage resources | Resource Group or Subscription |
| **User Access Administrator** | Create role assignments | Resource Group or Subscription |
| **Network Contributor** | Manage VNets, NSGs, route tables | Resource Group |

**Command to grant missing roles:**

```bash
RG_ID=$(az group show --name <resource-group> --query "id" -o tsv)

# Grant Contributor
az role assignment create \
  --assignee "$DEPLOYER_ID" \
  --role "Contributor" \
  --scope "$RG_ID"

# Grant User Access Administrator
az role assignment create \
  --assignee "$DEPLOYER_ID" \
  --role "User Access Administrator" \
  --scope "$RG_ID"
```

```bash
# register-providers.sh

providers=(
  "Microsoft.ApiManagement"
  "Microsoft.CognitiveServices"
  "Microsoft.MachineLearningServices"
  "Microsoft.KeyVault"
  "Microsoft.Network"
  "Microsoft.OperationalInsights"
  "Microsoft.Insights"
  "Microsoft.Authorization"
  "Microsoft.Storage"

for provider in "${providers[@]}"; do
  echo "Registering $provider..."
  az provider register --namespace "$provider" --wait
  state=$(az provider show --namespace "$provider" --query "registrationState" -o tsv)
  echo "  Status: $state"
done
```

| **2.10** Storage naming | `stcitadel<env><unique>` | `stcitadelprod001` | [ ] |

**Naming validation script:**

```bash
#!/bin/bash
# validate-naming.sh

ENV="$1"
UNIQUE_SUFFIX="$2"

validate_name() {
    local name="$1"
    local max_length="$2"
    
    if [ ${#name} -gt $max_length ]; then
        echo "❌ FAIL: '$name' exceeds $max_length characters"
        return 1
    fi
    
    if [[ ! "$name" =~ ^[a-z0-9-]+$ ]]; then
        echo "❌ FAIL: '$name' contains invalid characters (only lowercase, numbers, hyphens)"
        return 1
    fi
    
    echo "✅ PASS: '$name'"
}

echo "Validating resource names for environment: $ENV"
echo ""

validate_name "rg-citadel-hub-$ENV" 90
validate_name "apim-citadel-$ENV" 50
validate_name "kv-citadel-$ENV-$UNIQUE_SUFFIX" 24
validate_name "vnet-hub-$ENV" 64
```

## Validation Commands Quick Reference

### Complete Pre-Deployment Validation

```bash
#!/bin/bash
# pre-deployment-validation.sh

set -e

echo "========================================="
echo "AI Landing Zone Pre-Deployment Validation"
echo "========================================="
echo ""

# Configuration
SUBSCRIPTION_ID="$1"
RESOURCE_GROUP="$2"
LOCATION="$3"

if [ -z "$SUBSCRIPTION_ID" ] || [ -z "$RESOURCE_GROUP" ] || [ -z "$LOCATION" ]; then
    echo "Usage: $0 <subscription-id> <resource-group> <location>"
    exit 1
fi

echo "Subscription: $SUBSCRIPTION_ID"
echo "Resource Group: $RESOURCE_GROUP"
echo "Location: $LOCATION"
echo ""

# Set subscription
az account set --subscription "$SUBSCRIPTION_ID"

# Check 1: Subscription state
echo "1. Checking subscription state..."
STATE=$(az account show --query "state" -o tsv)
if [ "$STATE" = "Enabled" ]; then
    echo "   ✅ Subscription is Enabled"
else
    echo "   ❌ Subscription state: $STATE"
    exit 1
fi

# Check 2: Resource providers
echo "2. Checking resource providers..."
REQUIRED_PROVIDERS=(
    "Microsoft.ApiManagement"
    "Microsoft.CognitiveServices"
    "Microsoft.KeyVault"
    "Microsoft.Network"
)

for provider in "${REQUIRED_PROVIDERS[@]}"; do
    REG_STATE=$(az provider show --namespace "$provider" --query "registrationState" -o tsv)
    if [ "$REG_STATE" = "Registered" ]; then
        echo "   ✅ $provider: Registered"
    else
        echo "   ❌ $provider: $REG_STATE"
    fi
done

# Check 3: Resource group exists
echo "3. Checking resource group..."
if az group show --name "$RESOURCE_GROUP" &>/dev/null; then
    echo "   ✅ Resource group exists"
else
    echo "   ⚠️  Resource group will be created during deployment"
fi

# Check 4: Current user permissions
echo "4. Checking permissions..."
USER_ID=$(az ad signed-in-user show --query "id" -o tsv)
ROLES=$(az role assignment list --assignee "$USER_ID" --scope "/subscriptions/$SUBSCRIPTION_ID" --query "[].roleDefinitionName" -o tsv)
echo "   Assigned roles: $ROLES"

echo ""
echo "========================================="
echo "Validation Complete"
echo "========================================="
```

## Go/No-Go Criteria

### Critical Checks (Must Pass)

All of the following must pass before deployment:

- [ ] Azure subscription is **Enabled** and active
- [ ] User has **Contributor** and **User Access Administrator** roles
- [ ] All required resource providers are **Registered**
- [ ] Network CIDRs are planned with **no overlaps**
- [ ] APIM SKU is available in target **region**
- [ ] Cost estimate is **approved**

### Warnings (Document Mitigation)

The following may proceed with documented mitigation:

- [ ] Resource quotas near limit — Mitigation: Request increase before deployment
- [ ] New region for organization — Mitigation: Verify compliance requirements
- [ ] First AI Landing Zone deployment — Mitigation: Schedule additional validation time
