# 🔧 Security Implementation Guide

## Overview

This guide provides practical, step-by-step instructions for implementing security controls in AI Landing Zones. From initial deployment through ongoing hardening, follow these patterns to establish enterprise-grade security for your AI workloads.

## Security Validation

### Pre-Deployment Security Scan

```powershell
# Validate ARM template security
$templatePath = "./main.bicep"
$scanResults = Test-AzTemplate -TemplatePath $templatePath

# Check for security issues
$securityIssues = $scanResults | Where-Object { 
  $_.Issue.Category -eq "Security" 
}

if ($securityIssues) {
  Write-Error "Security issues found:"
  $securityIssues | ForEach-Object {
    Write-Host "  - $($_.Issue.Message) at $($_.Resource.ResourceName)"
  }
  exit 1
}

# Check for public endpoints
$publicEndpoints = $scanResults | Where-Object {
  $_.Resource.Type -eq "Microsoft.Network/publicIPAddresses"
}

if ($publicEndpoints) {
  Write-Warning "Public IP addresses detected. Ensure these are necessary:"
  $publicEndpoints | ForEach-Object {
    Write-Host "  - $($_.Resource.ResourceName)"
  }
}
```

### Post-Deployment Validation

```bash
#!/bin/bash
# Post-deployment security validation

echo "=== Security Validation ==="

# Check NSG rules
echo "Checking NSG rules..."
az network nsg list -g $RESOURCE_GROUP --query "[].{Name:name, Rules:securityRules[].{Name:name, Access:properties.access, Direction:properties.direction}}"

# Check private endpoints
echo "Checking private endpoints..."
az network private-endpoint list -g $RESOURCE_GROUP --query "[].{Name:name, Subnet:properties.subnet.id, NIC:properties.networkInterfaces[0].id}"

# Check Key Vault network ACLs
echo "Checking Key Vault network settings..."
az keyvault list -g $RESOURCE_GROUP --query "[].{Name:name, Network:properties.networkAcls.defaultAction}"

# Check storage encryption
echo "Checking storage encryption..."
az storage account list -g $RESOURCE_GROUP --query "[].{Name:name, Encryption:encryption.services.blob.enabled}"

# Check diagnostic settings
echo "Checking diagnostic settings..."
az monitor diagnostic-settings list --resource $(az apim list -g $RESOURCE_GROUP --query "[0].id" -o tsv) --query "[].name"

echo "=== Validation Complete ==="
```

```yaml
  query: |
```

```powershell
# Export activity logs
az monitor activity-log list --start-time $IncidentTime --offset 24h | 
  Out-File "$evidencePath/activity-logs.json"

# Export NSG flow logs
$nsgs = az network nsg list -g $ResourceGroup --query "[].name" -o tsv
foreach ($nsg in $nsgs) {
  az network nsg show -g $ResourceGroup -n $nsg | 
    Out-File "$evidencePath/nsg-$nsg.json"
}

# Export APIM logs
$apimName = az apim list -g $ResourceGroup --query "[0].name" -o tsv
az apim show -g $ResourceGroup -n $apimName | 
  Out-File "$evidencePath/apim-config.json"

# Upload to secure storage
az storage blob upload-batch --destination evidence --source $evidencePath 

Write-Host "Evidence collected to $StorageAccount/evidence/"
```

| Incident Type | Immediate Action | Recovery Steps |
|--------------|-----------------|----------------|
| **Compromised API Key** | Revoke key | Generate new key, update consumers, audit access |
| **Data Breach** | Isolate resource | Preserve evidence, notify stakeholders, remediate |
| **DDoS Attack** | Enable protection | Scale resources, analyze attack, implement filters |
| **Configuration Drift** | Restore from code | Redeploy from IaC, investigate cause |
