# Prerequisites

Before deploying the SOF1A 2.0 AI Governance Platform, ensure you have the necessary access, tools, and knowledge in place.

## Azure Requirements

### Subscription Access

<Check>Azure subscription with Owner or Contributor + User Access Administrator roles</Check>
<Check>Ability to create resource groups at the subscription level</Check>
<Check>Quota for required Azure services in target regions</Check>

### Required Azure Services

The following services will be provisioned. Ensure your subscription has quota available:

| Service | Purpose | Quota Consideration |
|---------|---------|---------------------|
| Virtual Networks | Network isolation | 1 VNet per landing zone |
| Azure OpenAI | AI model hosting | Token quota per region |
| Azure API Management | API gateway | Developer or Premium tier |
| Key Vault | Secret management | Standard tier |
| Log Analytics | Monitoring | Pay-as-you-go or commitment tier |
| Azure ML | Machine learning workspace | Standard tier |

## Tools Required

### Required Tools

<Steps>
  <Step title="Azure CLI">
    Install [Azure CLI](https://docs.microsoft.com/cli/azure/install-azure-cli) version 2.40 or later:
    ```bash
    az --version
    ```
  </Step>
  <Step title="Git">
    Install [Git](https://git-scm.com/downloads) for repository management:
    ```bash
    git --version
    ```
  </Step>
  <Step title="PowerShell (Windows)">
    Install [PowerShell 7+](https://docs.microsoft.com/powershell/scripting/install/installing-powershell) for deployment scripts:
    ```powershell
    $PSVersionTable.PSVersion
    ```
  </Step>
</Steps>

## Environment Validation

Before proceeding, validate your environment:

```bash
# Verify Azure CLI login
az account show

# List available subscriptions
az account list --output table

# Check resource provider registrations
az provider list --output table | grep -E "Microsoft.Network|Microsoft.ApiManagement|Microsoft.KeyVault|Microsoft.CognitiveServices|Microsoft.MachineLearningServices"
```

<Note>
  If resource providers are not registered, you may need to register them:
  ```bash
  az provider register --namespace Microsoft.ApiManagement
  ```
</Note>
